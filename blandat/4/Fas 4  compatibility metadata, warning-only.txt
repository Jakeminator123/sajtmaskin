Task:
Add warning-only compatibility metadata for dossiers/capabilities.

Goal:
Make dossier selection aware of scaffold/buildIntent/route context without blocking generation yet.

Important:
This phase should not block builds. It should only produce warnings.

Add metadata shape:
- compatibleScaffolds
- discouragedScaffolds
- incompatibleScaffolds
- buildIntents
- requiredRouteKinds
- requiredApiRoutes
- placementHint
- conflictsWithCapabilities
- severity: info | warn | block

Start with a few dossiers only:
- ai-chat
- payments/stripe-checkout
- visual-3d / three-fiber
- interactive-game
- contact-form
- auth

Examples:
ai-chat:
- preferred: app-shell, dashboard, base-nextjs
- allowed: landing-page, saas-landing
- placement: feature-route or shared-component
- may require api route

payments:
- preferred: ecommerce, saas-landing, app-shell
- warn if no pricing/cart/checkout route context

interactive-game:
- preferred: base-nextjs, app-shell
- discourage: blog, portfolio, ecommerce unless prompt explicitly asks for game

contact-form:
- mostly universal
- placement: section or contact route

auth:
- preferred: app-shell, dashboard, auth-pages, saas-landing
- route requirements: login/signup or auth flow

Constraints:
- No hard blockers in first version.
- Do not change dossier selection order unless warnings are explicitly non-breaking.
- Surface warnings in routing inspector/backoffice if available.

Tests:
- ai-chat on app-shell: no warning
- ai-chat on landing-page: info/warn, not block
- stripe checkout without checkout/pricing context: warn
- game on blog prompt: warn unless explicit playable game context
