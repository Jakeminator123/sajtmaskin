OpenClaw Setup Guide — Sajtagenten (infra/openclaw)
============================================================

Purpose
-------
This folder contains the Docker setup for "Sajtagenten", an OpenClaw-based
AI agent with broad read access to a target website. The agent can read pages,
sitemaps, metadata, and public APIs on the configured site.

This is a generic template. Adapt IDENTITY.md and environment variables
to your specific project before deploying.


1) Architecture
---------------
- The OpenClaw Gateway runs as its OWN standalone Docker Web Service
  (Render, Railway, etc.) — separate from the app deployment.
- The consuming app (Sajtmaskin on Vercel) runs separately and only ships a
  React widget + thin API proxy under src/app/api/openclaw/* and
  src/app/api/did/chat. It calls the gateway over HTTPS.
- App -> gateway link is just env: OPENCLAW_GATEWAY_URL +
  OPENCLAW_GATEWAY_TOKEN. The widget is gated by IMPLEMENT_UNDERSCORE_CLAW.
- The proxy always targets the single hardcoded agent "openclaw:sajtagenten"
  (see src/app/api/openclaw/chat/route.ts) — there is no per-agent env var.
- Sajtagenten reads the target site via configured URL (SAJTAGENT_TARGET_SITE_URL).
- To run a second/new gateway: clone this service to a new Render instance
  and repoint the app's OPENCLAW_GATEWAY_URL. No code change required.

-------------

3) Render Service Configuration
-------------------------------
Service type: Web Service
Runtime:      Docker
Root dir:     infra/openclaw

Disk:
- Mount path: /root/.openclaw
- Size: 1 GB or larger

Required environment variables (see render.yaml for the canonical stubs):
- OPENCLAW_GATEWAY_TOKEN=<secret>                  (shared with the app)
- OPENAI_API_KEY=<secret>
- SAJTAGENT_TARGET_SITE_URL=https://example.com    (site the agent reads)

Recommended / instance-specific:
- SAJTAGENT_ALLOWED_ORIGINS=https://my-app.vercel.app,https://staging.example.com
    Comma-separated browser origins allowed to reach the controlUi. The list
    is built entirely from this var + SAJTAGENT_TARGET_SITE_URL at boot;
    http://localhost:3000 is always included. No hostname is hardcoded, so a
    new deployment URL works without a code change.

Optional environment variables (have safe defaults):
- OPENCLAW_GATEWAY_PORT=10000
- OPENCLAW_GATEWAY_BIND=lan
- OPENCLAW_CONTROLUI_DISABLE_DEVICE_AUTH=false
- OPENCLAW_MODEL_PRIMARY=openai/gpt-5.1-codex
- OPENCLAW_MODEL_FALLBACK=openai/gpt-5.3-codex
- JUICEFACTORY_API_KEY=<secret, only for the Qwen provider>

Notes:
- Do not manually set PORT unless you have a specific reason.
- Port resolution order: PORT -> OPENCLAW_GATEWAY_PORT -> 18789


4) App (Vercel) Environment Variables
-------------------------------------
All three are required for the widget + Sajtagenten surfaces to activate
(src/lib/config.ts -> OPENCLAW.surfaceEnabled). Missing any one hides the
widget entirely.
- OPENCLAW_GATEWAY_URL=https://<your-gateway>.onrender.com  (this Render URL)
- OPENCLAW_GATEWAY_TOKEN=<same secret as the Render service>
- IMPLEMENT_UNDERSCORE_CLAW=true

WARNING: OPENCLAW_GATEWAY_URL must point at the separate gateway service —
never at the app's own host (http://localhost:3000 or the prod app URL),
or /api/openclaw/chat loops back onto itself and returns 404.

Note: there is NO per-agent env var. OPENCLAW_AGENT_ID / OPENCLAW_AGENT_TOKEN
were removed (2026-06) — they are not read anywhere in src/. The proxy always
targets the hardcoded agent "openclaw:sajtagenten".


5) Smoke Test Checklist
-----------------------
Render / hosting checks:
1) GET <gateway-url>/health -> HTTP 200
2) Logs contain:
   - "Config written"
   - "[gateway] listening on ws://0.0.0.0:<port>"

Frontend checks:
1) GET <app-url>/api/openclaw/health  -> healthy response
2) POST <app-url>/api/openclaw/chat   -> HTTP 200 with SSE chunks


6) Common Failures and Fixes
-----------------------------
Symptom: Build fails at npm install with SSH/git errors
Fix:     git + ca-certificates installed; GitHub SSH -> HTTPS rewrite in Dockerfile.

Symptom: "No open ports detected" on Render
Fix:     Ensure OPENCLAW_GATEWAY_BIND=lan and service listens on correct port.

Symptom: "Gateway start blocked: set gateway.mode=local"
Fix:     gateway.mode must be "local", or run with --allow-unconfigured.

Symptom: /v1/chat/completions returns 405
Fix:     gateway.http.endpoints.chatCompletions.enabled must be true.

Symptom: "No API key found for provider openai-codex"
Fix:     Set OPENAI_API_KEY on the hosting platform.


7) Security Notes
-----------------
- Never commit secrets to git.
- Rotate OPENCLAW_GATEWAY_TOKEN after stabilization.
- The agent has READ-ONLY access to the target site - it cannot modify content.


8) Customization
----------------
- Edit config/agents/sajtagenten/agent/IDENTITY.md to change agent personality.
- Add workspace skill files to config/workspace/ for domain-specific capabilities.
- Set SAJTAGENT_ALLOWED_ORIGINS to whitelist additional frontend origins.
- Set SAJTAGENT_TARGET_SITE_URL to the site the agent should be able to read.
