## Contract Risks
- Buy-credits page has duplicate package definitions that can drift from `lib/stripe.ts`.
- Pricing copy can drift from actual `getCreditCost` values.
- Checkout copy mentions Apple Pay/Google Pay while server only lists card unless Stripe handles wallets implicitly.
- 429 checkout has no dedicated UI handling.
- Webhook latency can make balance stale after success redirect.

## Required Adaptations
- Use one package source for buy-credits and landing pricing.
- Sync “what it costs” copy with `pricing.ts` and model catalog.
- Add 429 handling with `retryAfter`.
- If checkout/payment methods change in master, align frontend copy.
