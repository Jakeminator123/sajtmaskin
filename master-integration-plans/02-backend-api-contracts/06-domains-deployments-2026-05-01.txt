## Contract Risks
- `projectId` has mixed semantics: Vercel project id for link/verify and internal deployment id for save.
- `DEPLOY_MISSING_ENV` field names must stay aligned with `useBuilderDeployActions`.
- SSE status can lead DB list refresh; frontend treats stream status as current truth.
- `domains/save` errors are currently swallowed.
- `domains/check` is unauthenticated and rate-limited.
- SEO zod/precedence changes must match UI payload and preferences patch.

## Required Adaptations
- Preserve post-deploy toast, error-log persistence, and 409 handling.
- Update `useBuilderDeployActions` typings whenever deployment response/schema changes.
- If domains/save auth changes, add explicit UI feedback instead of silent catch.
- Keep list GET response shape stable if deployment history UI consumes it.
