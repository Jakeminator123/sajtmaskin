V0 PROMPT GUIDE - Best Practices
Source: https://v0.dev/docs/text-prompting
Last updated: 2026-01-29

================================================================================
CORE FRAMEWORK (3-Part Structure)
================================================================================

1. PRODUCT SURFACE (What)
   - Exact components, features, data
   - Example: "Dashboard displaying top 5 performers with names, revenue, trend chart"

2. CONTEXT OF USE (Who/When)
   - User role, device, decision being made
   - Example: "Sales managers checking during morning standups on desktop"

3. CONSTRAINTS & TASTE (How)
   - Style preferences, colors, layout, platform assumptions
   - Example: "Minimal dark mode, mobile-first, shadcn/ui components"

================================================================================
PROMPT ANATOMY (ICOD)
================================================================================

I - Instruction: What task to do
C - Context: Background information
O - Output Indicator: Format requirements
D - Data: The actual input

================================================================================
EXAMPLES - Good vs Bad
================================================================================

BAD:  "Make a website"
GOOD: "Create a portfolio website for a freelance designer with hero section,
       projects showcase, contact form, and responsive dark mode design"

BAD:  "Build an e-commerce platform with auth, catalog, cart, payments, orders,
       admin dashboard, and analytics" (too complex in one prompt)
GOOD: "Start with a product catalogue page showing products in a grid layout"
      Then: "Add a shopping cart with localStorage persistence"

================================================================================
PROMPT TEMPLATES
================================================================================

CRUD OPERATIONS:
"Create a resource management interface with:
- List view with sorting
- Add/edit forms with validation
- Delete confirmation modal
- Search and pagination"

AUTHENTICATION:
"Implement [auth method] with:
- Login/register forms
- Password reset flow
- Session management
- Protected routes"

DASHBOARD:
"Build a dashboard widget showing [metric] with:
- Current value display
- Trend indicator (up/down arrow)
- Comparison to previous period
- Interactive chart with hover details"

LANDING PAGE:
"Create a landing page with:
- Hero section with CTA
- Features grid (3-4 items)
- Pricing table (3 tiers)
- Testimonials carousel
- Footer with links"

================================================================================
ADVANCED TECHNIQUES
================================================================================

1. COMPONENT-FIRST:
   Build piece by piece, starting with individual components.
   "Create a header component with navigation and logo"
   Then: "Add a hero section below the header"

2. PROGRESSIVE ENHANCEMENT:
   Start simple, add features incrementally:
   1. "Create a basic todo list with add/delete"
   2. "Add localStorage to save todos"
   3. "Add completion checkboxes and filtering"

3. PLANNING FIRST (for complex projects):
   "Create a PRD for a customer support ticket system with agent dashboard"
   Then: "Build the MVP based on the PRD, starting with core features"

================================================================================
MODEL TIERS (v0 Platform API)
================================================================================

v0-mini: Fast, cheap, simple components
v0-pro:  Balanced quality/speed
v0-max:  Best quality, complex UIs, thinking enabled (RECOMMENDED)

================================================================================
MODEL API (v0 Model API / openai-compat)
================================================================================

v0-1.5-md: 128K context, standard quality (text model)
v0-1.5-lg: 512K context, premium quality (text model)

Used in this app for Prompt Assist when provider=openai-compat.
Note: Deep Brief Mode currently uses AI Gateway only.

================================================================================
PROMPT ASSIST (Your app's feature)
================================================================================

Your app automatically enhances prompts via AI Gateway:
- Provider: gateway (Vercel AI Gateway)
- Model: openai/gpt-5.2 (default)
- Fallbacks: claude-sonnet-4.5, gemini-2.5-flash

Alternate prompt assist:
- Provider: openai-compat (v0 Model API)
- Models: v0-1.5-md / v0-1.5-lg

The enhance step:
1. User writes simple prompt
2. Prompt assist rewrites with more detail
3. Enhanced prompt sent to v0

================================================================================
INTEGRATION WITH YOUR APP
================================================================================

Endpoints:
- /api/v0/chats/stream (create new chat)
- /api/v0/chats/{chatId}/stream (continue chat)
- /api/v0/chats/init (import from GitHub/ZIP)
- /api/ai/brief (deep prompt analysis)

Request body keys:
- message: The user prompt
- modelId: v0-mini | v0-pro | v0-max
- thinking: boolean (enable reasoning)
- imageGenerations: boolean (generate images)
- system: optional system prompt
- attachments: optional files
