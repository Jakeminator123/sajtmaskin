AC-SCHEMA - ENHETLIG AI/GATEWAY-LOGIK
Syfte: definiera slutligt beteende nar vi rensar och forenklar.

Malbild:
- Alla model- och assist-val sker endast i buildern.
- Inga personliga API-nycklar i UI eller DB.
- Samma beteende oavsett entry (home/wizard/category/audit).

AC - Flode
- [ ] Start/kategori skickar endast prompt till /builder (inga model-parametrar).
- [ ] Builder header ar enda stallet for model tier + prompt assist.
- [ ] "Forbattra" anvander /api/ai/chat eller /api/ai/brief.
- [ ] "Skicka" anvander /api/v0/chats/stream.
- [ ] Provider=off gor inga /api/ai/*-anrop.
- [x] v0-systemprompt skickas i v0-routes (system vidarebefordras).
- [x] Custom Instructions ar per chat (lagras lokalt per chatId).
- [x] Manuell "Forbattra" skippar auto-assist vid forsta skick.
- [x] imageGenerations styrs av v0-tillganglighet, Blob endast for deploy.

AC - Konfig
- [ ] Endast server-env anvands for API-nycklar.
- [ ] Tydlig feltext om AI Gateway nyckel saknas.
- [ ] Tydlig feltext om v0 Model API nyckel saknas (openai-compat).

AC - UX/Debug
- [ ] UI visar aktiv provider + modell + deep.
- [ ] Debug visar faktisk provider/model som anropas.
- [ ] Ingen dold fallback utan info.
- [x] Tooltip forklarar skillnaden mellan Model Tier och Prompt Assist.
- [x] Tool debug forklarar hasInput/hasOutput/state/toolCallId och visar post-check-sammanfattning.

Noter:
- Platform API (v0-sdk) bygger sajten (v0-mini/pro/max).
- Model API (AI SDK / @ai-sdk/vercel) anvander v0-1.5-md/lg for text, ej for build.
- CSS-validering auto-fixar Tailwind v4 fel efter generation; post-check ar read-only.
