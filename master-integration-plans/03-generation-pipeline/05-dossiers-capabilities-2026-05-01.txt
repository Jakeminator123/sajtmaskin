SAJTMASKIN MASTER INTEGRATION PLAN — 03 Generation Pipeline — 05-dossiers-capabilities
Date: 2026-05-01

## Master Pipeline Changes
- Dossiers live under `data/dossiers/hard|soft/<id>`.
- Strict schema: `docs/schemas/strict/dossier.schema.json`.
- CI validates manifests, imports, default uniqueness, headings, and verbatim files.
- Runtime registry is permissive and warns/defaults rather than strict AJV.
- Capability map is generated tooling/sanity, not runtime source.
- Dossier selection is deterministic by capability and default/id ordering.
- Orchestration merges brief, inferred capabilities, and caller follow-up capabilities.

## Local Behaviors To Preserve
- Permissive runtime load plus strict CI.
- Merge order: brief -> inferred bridge -> caller follow-up.
- Deterministic tie-break and duplicate default warnings.
- Env var `configured` state based on `process.env`.
- `SAJTMASKIN_DOSSIER_PIPELINE=false|0` opt-out.

## Merge/Port Plan
- Diff `data/dossiers/**`, strict schema, manifest validator, orchestrate inference, follow-up vocabulary, and scripts.
- After dossier additions, run validation and capability-map check/write.
- If master adds runtime AJV, explicitly decide permissive vs strict runtime behavior.
