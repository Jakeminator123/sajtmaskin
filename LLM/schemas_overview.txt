SAJTMASKIN SCHEMAS - OVERVIEW
Purpose: Samla en standard for hur anrop och floden fungerar.
Scope: Endast beskrivande dokument (ingen runtime).

Files in this folder:
- `LLM/scaffold-schema.txt` - human-readable contract for internal scaffold design and usage
- (No standalone API call doc yet. API flows are documented inline in the source code.)

Relation to other docs:
- src/...: kodkalla for sanning, schemas maste uppdateras nar kod andras.
- `LLM/egen-motor/scaffold-status-and-plan.md`: roadmap/status for scaffold expansion

How to update:
0) Uppdatera endast pa uttrycklig begaran.
1) Nar post-generation checks utokas (CSS/unicode) -> uppdatera post-generation-checks.txt.

Terminologi:
- gen/ pipeline (default): Egen kodgenereringsmotor via OpenAI GPT 5.2 + AI SDK.
  Entry: src/lib/gen/fallback.ts createGenerationPipeline().
  Engine: src/lib/gen/engine.ts (streamText + createCodeGenSSEStream).
  Postprocessering: src/lib/gen/suspense/ (TransformStream-regler).
  Autofix: src/lib/gen/autofix/ (import-validering, JSX-check, dep-komplettering).
  Se src/lib/gen/README.md for arkitektur.
- Platform API (v0-sdk) = fallback kodgenerering (V0_FALLBACK_BUILDER=y).
  Canonical tiers: v0-max-fast (default), v0-1.5-lg, v0-1.5-md, v0-gpt-5.
  Legacy aliases: v0-mini -> v0-1.5-md, v0-pro -> v0-1.5-md, v0-max -> v0-max-fast.
  Source of truth: src/lib/v0/models.ts
- Model API (AI SDK / @ai-sdk/openai) = textmodell (v0-1.5-md/lg).
  Anvands enbart for prompt assist, ej for sajtbygge.

Designprincip:
- Kort och exakt. Peka till kod, duplicera inte logik.
- Inga hemligheter eller nyckelvarden i texten, men kan skrivas i chatten och användas fritt där och i "env"-filer.
- Teknisk standard och sprakregel ligger i DEFAULT_CUSTOM_INSTRUCTIONS (src/lib/builder/defaults.ts).

Wizard enrich (2026-03, V3):
- Endpoint: POST /api/wizard/enrich
- Request:
  - mode: "step" | "final_check" (default "step")
  - step: 1..5
  - data:
    - companyName, industry, location, existingWebsite
    - inspirationSites[]
    - purposes[], targetAudience, usp, selectedVibe, specialWishes
    - previousFollowUps: Record<string,string>
    - companyLookup?: { found, employees?, revenueKsek?, industries?, purpose? }
    - competitors?: { name, website? }[]
  - scrapeUrl?: string
- Response:
  - questions[]: { id, text, type, options?, placeholder?, priority?, dependsOn? }
  - suggestions[]: { type, text }
  - insightSummary?: string | null
  - meta?: { confidence?, needsClarification?, unknowns?, priority? }
  - scrapedData?: { title?, description?, headings?, wordCount?, hasImages?, textSummary? }
  - contextHash?: string
- Not:
  - mode=final_check anvands for AI-klargoranden innan slutbrief i wizardn.
  - previousFollowUps ska alltid beaktas for att undvika repetitiva fragor.
  - companyLookup och competitors injiceras i AI-kontexten for smartare fragor.

Wizard company-lookup (2026-03, V3):
- Endpoint: POST /api/wizard/company-lookup
- Request: { companyName: string, orgNr?: string }
- Response: CompanyLookupResult { found, companyName?, orgNr?, companyType?, city?,
    address?, industries?, revenueKsek?, employees?, ceo?, homepage?, purpose?,
    source?: "allabolag" | "ai_search" | "none" }
- Strategi: 1) Cheerio-skrapa allabolag.se 2) AI web_search fallback 3) Tomt svar
- Trigger: Automatiskt fran steg 1 nar companyName >= 3 tecken (debounce 1200ms)

Wizard competitors (2026-03, V3):
- Endpoint: POST /api/wizard/competitors
- Request: { companyName, industry, location?, existingWebsite? }
- Response: { competitors[]: { name, description, website?, lat?, lng?, isInspiration? },
    marketInsight?: string }
- Trigger: Automatiskt fran steg 1 nar companyName + industry ifyllt (debounce 1500ms)
