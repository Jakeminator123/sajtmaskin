# 818 — svärm → beslut → ändring → review

**Kort orkestrering** (ofta 10–30 min): åtta parallella **read-only** subagenter samlar in korta bedömningar (gärna **%** där det passar) om **samma** fråga från olika vinklar. Du (huvudagenten) **sammanfattar i en tabell**, fattar **ett** beslut, genomför **minimal** ändring, kör **verifiering**, och låter **1–2** agenter (eller samma 8 i “review-läge”) **granska diffen** innan användaren committar.

**Inte samma som** `/långbänk` (lång, många spår, tung modellval). 818 = **en** fråga, **många korta röster**, snabb **go/no-go**.

## När användaren kör `/818` (med eller utan @-underlag)

1. **Förstå frågan** i en mening. Om scopet är tomt: be om en rad om vad som ska avgöras.
2. **Lansera 8 Task-agenter parallellt** i **en** assistant-turn (`Task`, flera anrop):
   - `subagent_type`: `explore`
   - `readonly`: `true`
   - `model`: `composer-2` eller `composer-2-fast` (snabb inventering / risk-%).
   - Varje agent får **en distinkt vinkel** (ex.: kodrisk, doc/kanon, regressionsrisk, namnskugga, drift mot glossary, testlucka, säkerhet, “vad kan vi missat?”).
   - Kräv output: **max ~10 rader** per agent — tabell eller bullets med **% eller R/H/M/L** där det är lämpligt; **inga** noveller.
3. **Om du är osäker** på att workspace saknar fakta (t.ex. `.git` ej synlig för subagent): **du** kör shell/`Read`/`Grep` och **ersätter** subagentns gissning med verifierad sanning innan beslut.
4. **Sammanfatta** i **en** tabell (agent × bedömning × ev. %) och skriv **ett** rekommenderat beslut (implementera / vänta / bara radera gren / m.m.).
5. **Implementera** bara om beslutet är **glasklart** och **smalt**. Annars: lista öppna kontroller och **implementera inte**.
6. **Efter ändring**: kör `npm run typecheck`, relevanta `vitest`-filer om kod ändrats, `ReadLints` på ändrade filer.
7. **Review-pass**: lansera **1–2** `readonly` agents (`composer-2`) med prompt: “Review **only** the listed diff intent — godkänn / lista konkreta brister / regressionsrisk”. Alternativ: **fyra** agenter om ändringen är känslig. **Skriv inte ny kod i review-pass.**

## Anti-mönster

- Köra 818 utan parallella Task-anrop (missar poängen).
- Alla 8 med **samma** prompt (duplicerar svar — olika vinklar är måste).
- Subagents med skrivrätt för audit-fasen (`readonly: false` på läsfas).
- Bredda scope mitt i körning — säg “sidospår till nästa /818”.
