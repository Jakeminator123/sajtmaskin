v0.app – Gratis vs Betalt
1. Gratisanvändning (Free-plan)
•	Pris: Gratisplanen kostar \$0/mån. Du får en $5 inkl. månadskredit (virtuella krediter för API-anrop) varje månad[1].
•	Meddelandenbegränsning: Gratisanvändare kan göra upp till 7 chattförfrågningar per dag (”messages/day”)[1]. Detta motsvarar att du kan ge ~7 promptar till AI-assistenten varje dygn. (Vid överskridande krävs uppgradering eller väntan på ny dygnscykel.)
•	Kod och komponenter: Du kan generera obegränsat med UI-komponenter och kodfragment i dialogen. All genererad React/Tailwind-kod kan kopieras direkt från chattgränssnittet eller laddas ner som filer via API. V0 använder “Blocks” som kan förhandsgranskas och därefter klistras in i ditt projekt eller installeras med verktyg som shadcn CLI[2]. De genererade komponenterna är dina att använda fritt i egna projekt (se avsnitt om licens nedan).
•	Funktioner: Gratisplanen inkluderar Design Mode (visuell redigering) och möjlighet att synka med GitHub[3]. Du kan också direkt publicera (deploy) dina projekt till Vercel utan extra kostnad.
•	Begränsningar: Vissa avancerade funktioner kräver betalplan (se nedan). Till exempel är alla API-integrationer och större AI-modeller låsta till Premium/Team-planer[4][5].
2. Betalplaner och prissättning
•	Premium: Kostar \$20/mån per användare. Ger \$20 inkl. krediter per månad och \$2 dagliga gratiskrediter vid inloggning[6]. Utöver det kan du köpa extra krediter vid behov. Premium-planen har 5× större filbilagsgränser och stöd för Figma-import[6]. Modellen v0-1.5-lg (större AI-modell) och v0:s API ingår för Premium och uppåt.
•	Team: \$30/användare/mån. Inkluderar \$30 krediter per användare och \$2 dagliga krediter per användare[7]. Delade resurser för team (centraliserad fakturering, delade chattar/projekt). Krävs för att samarbeta flera personer och dela projekt.
•	Business: Cirka \$80–100/användare/mån (vanligtvis \$80 vid rea)[8]. Motsvarar team-plan med extra företagsfunktioner (SAML SSO, högre sekretess, träningsdatalöst). Fokuserar på säkerhet och stöd.
•	Enterprise: Anpassad prissättning (kontakta Vercel)[9]. Erbjuder bland annat opt-out för model-träning, SLA, specialsupport.
•	Modell-API (token-prissättning): Om du använder v0:s AI-modeller via API betalar du per token enligt prislistan. Exempel: v0 Agent (standardmodell) kostar \$1.5 per 1M input-tokens och \$7.5 per 1M output-tokens[10]. Den större modellen v0-1.5-lg kostar upp till \$15/\$75 per 1M (input/output)[11]. Dessa kostnader drar från dina krediter eller debiteras separat vid betalplan.
•	Kräver betalplan: För åtkomst till v0:s API (både modell-API och plattforms-API) krävs minst Premium- eller Team-plan[4][5]. Gratisplanen tillåter alltså endast webbgränssnittet, ej egna API-anrop.
3. Kodanvändning och licensiering
•	Användningsrätt: All kod och alla komponenter som genereras av v0 tillhör dig. Enligt Vercels villkor tilldelas (överlåts) eventuella rättigheter i AI-genererat innehåll till användaren[12]. Du får alltså fritt kopiera, ladda ner och använda genererad React/Next.js/Tailwind-kod i egna projekt, inklusive kommersiella sådana.
•	Granskning och ansvar: Vercel tillhandahåller ingen garantin för AI-kodens kvalitet eller originalitet. Deras villkor poängterar att output ”may not be bug-free” och bör granskas innan användning[13]. De rekommenderar att du kontrollerar kodsnuttar för fel och eventuella licenskonflikter innan produktion, särskilt för affärsbruk[14][12].
•	IP och unika resultat: Observera att AI:n kan generera liknande kod för olika användare. Du ansvarar själv för att säkerställa att koden inte inkräktar på andras upphovsrätt eller immateriella rättigheter[12][14]. Kort sagt: koden är din att använda, men gör alltid en egen granskning före lansering.
•	Begränsningar: Inga funktionella begränsningar i koden förutom ovan nämnda. Utdata kan dock ibland kräva anpassning. Det finns inga “stöldskyddande” hinder – allt genererat material kan exporteras från v0 och användas fritt i egna projekt.
4. API och integrationer
v0 erbjuder två huvudsakliga API: Model API (för kodgenerering) och Platform API (helhetsplattform). Båda kräver Premium/Team-prenumeration[4][5].
•	Model API: Liknar OpenAI Chat Completions. Du gör HTTP-anrop (t.ex. POST https://api.v0.dev/v1/chat/completions) med din prompt, och får AI-genererad kod eller text i respons. Stöder text- och bild-input (multimodal) och streaming-svar[15][16]. Modellerna (v0-1.5-md/lg) är optimerade för webbframkant (React, Next.js etc)[17]. Du kan exempelvis använda AI SDK:

 	import { generateText } from 'ai';
import { vercel } from '@ai-sdk/vercel';

const { text: code } = await generateText({
  model: vercel('v0-1.5-md'),
  prompt: 'Generate a Next.js page with a navbar and hero section',
});
console.log(code);
 	Detta ger dig React/Next.js-kod som du sedan kan infoga i din webbapp[18].
•	Platform API (v0 SDK): Ett REST-API för att hantera projekt, chattar, meddelanden och deploys. Med hjälp av @v0-sdk i t.ex. Node/Next.js kan du automatisera hela workflowen[19][20]. Du kan skapa nya projekt, initiera en chatt med ett prompt, och sedan få tillbaka genererade filer och förhandsvisningar. Exempel med @v0-sdk och v0:s plattforms-API:

 	// Skapa en ny chatt som genererar kod
const chat = await v0.chats.create({
  message: 'Skapa en todo-app med React och TypeScript',
});
// Läs av alla genererade filer
chat.files?.forEach(file => {
  console.log(file.name, file.content);
});
 	I detta exempel skriver chat.files ut varje filnamn och innehåll genererat av AI:n[21]. Dessa filer kan du spara i din applikation och köra.
•	Inbäddning och gränssnitt: Du kan även låta användare interagera med AI:n direkt på din egen sajt. Till exempel returnerar v0.chats.create() även en chat.demo-URL som visar chatten i en iFrame[22]. Ett enkelt React-exempel:

 	const chat = await v0.chats.create({ message: 'Skapa en responsiv navbar' });
return <iframe src={chat.demo} width="100%" height="600" />;
 	Detta bäddar in ett live-gränssnitt där AI:n genererar komponenter i realtid.
•	Funktioner och integrationer: Plattform-API:et stöder också externa verktyg och API-integrationer via “AI Tools”. Du kan t.ex. ge AI:n möjlighet att göra funktionanrop, läsa/skriva till databas eller anropa externa API:er under genereringen[23]. V0:s chattar kan med andra ord användas för avancerade arbetsflöden, automatiseringar och integrationer (kontinuerlig utveckling, CI/CD, chatbot-lösningar, mm.)[24][20].
5. Innehållstyper och outputs
V0 är designat för att generera gränssnitts-komponenter och webbapplikationer. Typiskt kan du få ut:
•	UI-komponenter: Enskilda React/Tailwind-komponenter, t.ex. formulärfält, knappar, navbarer, ikon- eller grafiksektioner. Dessa kan skräddarsys helt via din prompt.
•	One-pagers / landningssidor: Färdiga sidor som hero-sektioner, produktpresentationer, prisplaner, kontaktformulär med mera – hela layoutsektioner i en och samma kodgenerering[25]. V0 har många färdiga mallar för landningssidor som du kan använda som utgångspunkt (t.ex. SaaS-sidor, portföljer etc.)[26].
•	Dashboards och administrationsgränssnitt: Fullständiga vyer med flera komponenter och diagram, designade för att hantera data. Mallen är anpassad för instrumentpaneler (charts, tabeller etc.). Dokumentationen nämner uttryckligen stöd för dashboards, e‑handels¬sidor och data‐visualisering som vanliga exempel[25].
•	Fullstack-appar (MVP): V0 kan också skapa flerkomponents‐appar med routing, autentisering och API-integrationer. Även om kärnan är frontend-fokus, kan du låta AI:n generera grundläggande backend‐logik eller fetch-anrop (t.ex. enkla login-simuleringar och datafrågor) vid behov[27]. Notera dock att djupare backendfunktionalitet (t.ex. komplex databashantering) ofta kräver manuell efterbearbetning.
•	Tema och designsystem: Utöver kod kan du definiera färgscheman, typsnitt och komponentbibliotek (design systems) som återanvänds i projekten[28].
•	Exempelsidor och mallar: V0-sajten innehåller ett bibliotek med hundratals exempelprojekt (apps, mallar och komponenter) som communityn byggt – allt från spel och chatt‐appar till e‑handelslayouter och bloggar[29][25]. Dessa kan användas direkt eller inspirera dina egna förfrågningar.
6. Exempel: Generera React-sidor via API och integrera
Rent praktiskt fungerar arbetsflödet ungefär så här: du installerar v0:s SDK och gör ett API-anrop från din applikation eller arbetsmiljö, sedan infogar du resultatet i din kodbas.
Steg-för-steg-illustration:
1. Skaffa en API-nyckel i ditt v0-konto (kräver Premium/Team) och installera SDK (t.ex. @v0-sdk eller OpenAI-kompatibelt AI-SDK)[4].
2. Anropa v0:s API med din prompt. Exempel med v0:s plattforms-API (Node.js):

import { v0 } from 'v0-sdk';

// Skapa en ny chatt som bygger en sida
const chat = await v0.chats.create({
  message: 'Generera en Next.js-bloggsida med header och lista över artiklar',
});

// Hämta de genererade filerna
chat.files?.forEach(file => {
  console.log(`Fil: ${file.name}`);
  console.log(file.content); // Innehållet, t.ex. React-komponenter
});
Detta skriver ut filnamn och kod för varje fil AI:n skapade (t.ex. pages/index.tsx, komponentfiler, CSS). Du kan sedan spara dessa filer i ditt Next.js-projekt och köra sidan direkt[21].
3. Alternativt kan du använda Model API med AI-SDK för att enbart få kodtext tillbaka. Exempel:

import { generateText } from 'ai';
import { vercel } from '@ai-sdk/vercel';

const { text: code } = await generateText({
  model: vercel('v0-1.5-md'),
  prompt: 'Generate a React component for a user registration form',
});
console.log(code);
Variabeln code innehåller nu exempelvis JSX-kod för en registreringskomponent. Du kan klistra in detta i din egen .tsx-fil. (Tänk på att om du har flera filer eller större struktur kan plattforms-API vara enklare.)
4. Integrering i webbplats: Efter att ha fått koden kan du infoga den i din webbplats kodbas. Du kan också låta v0 generera hela sidor dynamiskt från din frontend. Exempel på inbäddat gränssnitt (iframe) via chat.demo:

const chat = await v0.chats.create({
  message: 'Skapa en responsiv navigationsbar med Tailwind',
});
return <iframe src={chat.demo} width="100%" height="400" />;
Detta bäddar in v0:s chattgränssnitt där AI:n i realtid visar hur navbaren byggs. Sedan kan du enkelt kopiera genererad kod direkt från chatten[22].
5. Efterarbete: Ofta vill man redigera eller förbättra koden manuellt. V0s “Design Mode” kan användas för visuell justering, eller så bearbetar du koden i din IDE. När koden sitter på plats kan du distribuera den till t.ex. Vercel (som v0 stödjer med ett klick)[30][20].
Sammanfattning: Genom API:et kan du alltså automatisera hela kedjan: skicka en beskrivning, ta emot React-/Next.js-kod och sedan förhandsvisa eller publicera den på din egen domän. V0 SDK för React (@v0-sdk/react) erbjuder även komponenter för att strömma chattmeddelanden live i UI om du vill bygga interaktiva AI-drivna verktyg[31]. Allt som krävs är en giltig API-nyckel och ett Premium-konto[4].
