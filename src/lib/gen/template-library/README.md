# Template Library (cursorignored generated data)

This directory is cursorignored because the generated JSON files are large.
Agents can read this README to understand the contents without loading them.

## Files

| File | Size | What it is |
|------|------|-----------|
| `template-library.generated.json` | ~6 MB | 64 curated template entries with metadata, quality scores, strengths, stack tags, and `selectedFiles` (code excerpts from dossiers). Generated by `config/scripts/build-template-library.ts` from `scaffold-pipeline/` (discovery + dossiers). |
| `template-library-embeddings.json` | ~3 MB | OpenAI `text-embedding-3-small` vectors for each template entry. Used by `search.ts` for semantic search at runtime. |
| `catalog.ts` | small | Loads and caches entries from the generated JSON. |
| `search.ts` | small | `searchTemplateLibrary(query)` — keyword + embedding search. `selectTemplateReferenceFiles(entry)` — picks code snippets to inject into system prompt. |
| `types.ts` | small | TypeScript types for entries, search results, selected files. |
| `embeddings-core.ts` | small | Embedding generation utilities. |
| `index.ts` | small | Barrel export. |

## Runtime flow

1. `system-prompt.ts` calls `rankTemplateReferences(prompt, scaffold)` which calls `searchTemplateLibrary(prompt)`
2. Top 3 matches are injected as "Relevant Template References" in the dynamic system prompt
3. Top 2 matches also get their `selectedFiles` (code excerpts) injected as "Reference Code Snippets"

## Regeneration

Run `npm run template-library:build` to rebuild from dossiers in `research/external-templates/reference-library/`.
Run `npm run template-library:embeddings` to rebuild embeddings.
