## Production Changes In Master
- Error-log RAG is development-only through config.
- Version error-log route persists production-relevant DB errors.
- RAG NDJSON/indexer is local file based.
- Metrics route is token-gated.

## Local Behavior To Preserve
- TF-IDF error-log RAG block in dynamic prompt.
- Verifier appends error log events.
- Admin frontlogs and prompt dumps.
- `/api/metrics` with prom-client metrics.

## Security / Secret Risks
- Metrics token.
- Admin frontlogs must require admin.
- Version error-log must stay tenant-scoped.
- RAG NDJSON may contain sensitive prompts/errors.

## Required Deploy / Observability UI

- VersionDiagnosticsDialog/version error logs.
- Backoffice observability pages.

- Treat prod RAG as infra/product decision, not just env.
