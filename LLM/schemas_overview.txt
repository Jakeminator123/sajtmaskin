SAJTMASKIN SCHEMAS - OVERVIEW
Purpose: Samla en standard for hur anrop och floden fungerar.
Scope: Endast beskrivande dokument (ingen runtime).

Files in this folder:
- `LLM/scaffold-schema.txt` - human-readable contract for internal scaffold design and usage

Relation to other docs:
- src/...: kodkalla for sanning, schemas maste uppdateras nar kod andras.
- `LLM/egen-motor/scaffold-status-and-plan.md`: roadmap/status for scaffold expansion
- `LLM/egen-motor/MOTOR-STATUS.md`: motor-arkitektur, modeller, pipeline

How to update:
0) Uppdatera endast pa uttrycklig begaran.
1) Nar post-generation checks utokas -> uppdatera post-generation-checks.txt.

Terminologi:
- gen/ pipeline (default): Egen kodgenereringsmotor via OpenAI.
  4 modell-tiers: Fast (gpt-4.1), Pro (gpt-5.3-codex), Max (gpt-5.4), Codex Max (gpt-5.1-codex-max).
  Entry: src/lib/gen/fallback.ts createGenerationPipeline().
  Engine: src/lib/gen/engine.ts (streamText + createCodeGenSSEStream).
  Post-gen: src/lib/gen/stream/finalize-version.ts (deduplicerad autofix+merge+save).
  Autofix: src/lib/gen/autofix/ (import-validering, JSX-check, dep-komplettering, scaffold-import-checker).
  Suspense: src/lib/gen/suspense/ (TransformStream-regler).
  Scaffolds: src/lib/gen/scaffolds/ (9 scaffolds, matcher, serialize).
  Se src/lib/gen/README.md for arkitektur.
- Platform API (v0-sdk) = fallback kodgenerering (V0_FALLBACK_BUILDER=y).
  Canonical tiers: v0-max-fast, v0-1.5-lg, v0-1.5-md, v0-gpt-5.
  Legacy aliases: v0-mini -> v0-1.5-md, v0-pro -> v0-1.5-md, v0-max -> v0-max-fast.
  Source of truth: src/lib/v0/models.ts
  OBS: Tier-ID:n borjar med "v0-" av historiska skal -- de anvands av bade v0-fallback
  och egen motor. I egen motor mappas de till OpenAI-modeller via v0TierToOpenAIModel().
- AI Gateway = prompt assist, deep brief, audit, wizard, domain suggestions.
  Nyckel: AI_GATEWAY_API_KEY eller VERCEL_OIDC_TOKEN (eller KV_REST_API_*).
  Gateway-modeller: openai/gpt-5.4, anthropic/claude-opus-4.6,
    anthropic/claude-sonnet-4.6, openai/gpt-5.2, openai/gpt-5.3-codex,
    openai/gpt-4.1-mini.
  Fallback-ordning: gpt-5.4 -> claude-opus-4.6 -> claude-sonnet-4.6 -> gpt-5.2.
  Tva typer av prompt assist:
    1. "polish" (Forbattra-knappen i chattrutan) = alltid gpt-4.1-mini (snabb/billig)
    2. "rewrite" (forbattrar inskickad prompt) = anvandarens valda modell (default gpt-5.3-codex)

Scaffolds (9 st + 1 fallback):
1. base-nextjs -- minimal starter
2. landing-page -- company/service landing
3. saas-landing -- SaaS product marketing
4. portfolio -- creatives, photographers
5. blog -- editorial, articles
6. dashboard -- admin, analytics
7. auth-pages -- login, register, reset
8. ecommerce -- webshop, product catalog
9. content-site -- broad content fallback
10. app-shell -- broad app fallback

Modellmappning (egen motor, standard):
  Fast (v0-max-fast) -> gpt-4.1
  Pro (v0-1.5-md) -> gpt-5.3-codex [default]
  Max (v0-1.5-lg) -> gpt-5.4
  Codex Max (v0-gpt-5) -> gpt-5.1-codex-max

Designprincip:
- Kort och exakt. Peka till kod, duplicera inte logik.
- Inga hemligheter i texten.
- Teknisk standard i DEFAULT_CUSTOM_INSTRUCTIONS (src/lib/builder/defaults.ts).

Wizard enrich (2026-03, V3):
- Endpoint: POST /api/wizard/enrich
- Request:
  - mode: "step" | "final_check" (default "step")
  - step: 1..5
  - data:
    - companyName, industry, location, existingWebsite
    - inspirationSites[]
    - purposes[], targetAudience, usp, selectedVibe, specialWishes
    - previousFollowUps: Record<string,string>
    - companyLookup?: { found, employees?, revenueKsek?, industries?, purpose? }
    - competitors?: { name, website? }[]
  - scrapeUrl?: string
- Response:
  - questions[]: { id, text, type, options?, placeholder?, priority?, dependsOn? }
  - suggestions[]: { type, text }
  - insightSummary?: string | null
  - meta?: { confidence?, needsClarification?, unknowns?, priority? }
  - scrapedData?: { title?, description?, headings?, wordCount?, hasImages?, textSummary? }
  - contextHash?: string

Wizard company-lookup (2026-03, V3):
- Endpoint: POST /api/wizard/company-lookup
- Request: { companyName: string, orgNr?: string }
- Response: CompanyLookupResult { found, companyName?, orgNr?, ... }

Wizard competitors (2026-03, V3):
- Endpoint: POST /api/wizard/competitors
- Request: { companyName, industry, location?, existingWebsite? }
- Response: { competitors[], marketInsight? }
