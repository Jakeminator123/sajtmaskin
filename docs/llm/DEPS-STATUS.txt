ENV-VARIABLER (KRÄVS)
---------------------
Databas:
- POSTGRES_URL (Supabase connection string)
- DB_SSL_REJECT_UNAUTHORIZED (default true; sätts false bara för lokal dev)

Auth/Säkerhet (KRÄVS I PROD):
- JWT_SECRET (kritiskt — utan denna loggas varning och all auth misslyckas)
- BACKOFFICE_PASSWORD (backoffice-auth)

AI / Kodgenerering:
- OPENAI_API_KEY (kravs nar V0_FALLBACK_BUILDER ej satt): gen/ engine
  (gpt-5.3-codex default, gpt-5.4 max), transcribe, template embeddings
- V0_FALLBACK_BUILDER (optional): sätt till "y" för att använda v0 Platform API
  istället för gen/ engine. Default: ej satt = ny motor
- V0_API_KEY (krävs när V0_FALLBACK_BUILDER=y): v0 Platform API för site generation.
  Används även för templates, registry init, download även utan fallback
- Canonical build-profile schema dokumenteras i `docs/schemas/model-build-profiles.md`
  (`fast`, `pro`, `max`, `codex`)
- AI_GATEWAY_API_KEY eller VERCEL_OIDC_TOKEN (prompt assist via gateway)
- BRAVE_API_KEY (Brave Web Search för company-lookup, competitors)
- USE_RESPONSES_API=true (opt-in OpenAI Responses API migration)
- AUDIT_WEB_SEARCH=false (opt-in web_search i audit-route)

Rate Limiting (REKOMMENDERAT I PROD):
- UPSTASH_REDIS_REST_URL eller KV_REST_API_URL (Upstash Redis REST)
- UPSTASH_REDIS_REST_TOKEN eller KV_REST_API_TOKEN
- Koden kollar bada namnkonventionerna (Vercels integration satter KV_REST_API_*)
- Not: Utan Redis faller rate limiting tillbaka till in-memory (ej palitligt i serverless)
- Alla Redis-nycklar har REDIS_KEY_PREFIX ("dev:" / "prod:") for miljo-isolering

Övrigt:
- BLOB_READ_WRITE_TOKEN (Vercel Blob for bilder)
- REDIS_URL (caching via ioredis)
- KV_URL (fallback for REDIS_URL)
- UNSPLASH_ACCESS_KEY (Unsplash stock-bilder i genererade sajter)
- CRON_SECRET (skyddar cron-endpoints)
- KOSTNADSFRI_API_KEY (API-nyckel for kostnadsfri-tjansten)
- Se ENV.md for fullstandig infra-topologi
