# Remediation pack för own-engine-flödet i sajtmaskin

## Executive summary

Du bad om en **körbar (runnable), split-file remediation plan** som en agent i **Cursor** kan exekvera för att refaktorera own-engine-flödet i repot `sajtmaskin` (branch `master`), med fokus på att:

- extrahera delad route-logik till en **OwnEngineBuildSession**-service,
- ersätta duplicerad meta/contract/plan-mode-kod med helpers,
- göra finalize **transaktionell** (undvika orphan assistant messages),
- rensa bort oanvända `STREAM_RESOLVE_*` och fixa/ta bort oanvänd `modelId`-parameter i `createOwnEnginePlanModeResponse`,
- byta namn `fallback.ts` → `generation-pipeline.ts` (med migration wrapper),
- tajta stream payload-typer samt lägga **golden tests** för stream-scenarion,
- behålla en tydlig **legacy v0-provider path** (extern provider) men separera den från own-engine-session,
- uppdatera/testa lokalt utan externa infra-antaganden.

**Historisk kontext (ursprungligt svar):** Skribenten saknade då live repo-åtkomst, så planen är discovery-baserad. **I Cursor med checkout:** kör sökning i `src/app/api/**` och `src/lib/**` mot symbolerna nedan och anpassa patcharna till faktiska filer.

Därför är detta ett **agent-exekverbart “remediation pack”** som:

1) automatiserar säkra, mekaniska förändringar (rename/wrappers, borttag av uppenbart död kod, scaffolding), och  
2) genererar modulära TypeScript-filer och test-harness med **explicit markerade “UNSPECIFIED / TODO: wire to repo”**-punkter samt robusta sökheuristiker.

För streaming och route handlers utgår strukturen från **Next.js Route Handlers** och vanlig streaming-response-arkitektur.  
För testbarhet föreslås att agenten kan köra lokalt med Node-baserad test runner eller Vitest (detaljer nedan).

## Utgångsläge, begränsningar och discovery-strategi

Eftersom repo-innehållet inte är läsbart här, bygger planen på en discovery-kedja som agenten kan köra direkt i repot:

- Sök efter route handlers för create och follow-up under typiska Next.js App Router-sökvägar (`app/api/.../route.ts` eller `src/app/api/.../route.ts`).  
- Sök efter:
  - `createOwnEnginePlanModeResponse`
  - `finalizeAndSaveVersion` (eller motsvarande finalize-funktion)
  - `builder-stream-contract.ts` (eller fil som definierar eventdelar och SSE-format)
  - `STREAM_RESOLVE_MAX_ATTEMPTS` / `STREAM_RESOLVE_DELAY_MS`
  - `src/lib/gen/fallback.ts` (eller annan “fallback” entry point)

Den körbara patchfilen jag ger nedan implementerar denna discovery som robusta heuristiker.

## Målarkitektur och nytt runtime-flöde

Målet är att routes blir tunna och deterministiska, med tydlig separation:

- Route (HTTP/Next) gör: parse + auth/guard + provider-val + session-start.
- `OwnEngineBuildSession` gör: kontrakts-gating + plan-mode + stream-start + finalize hook.
- En tydlig adapter för legacy v0-provider finns kvar, men isolerad bakom provider-resolver.
- Finalize görs tvåfasigt: **prepare in-memory → persist atomiskt** (eller pending-markering).

Streaming-svar i Next.js Route Handlers är direkt stödd (Response/ReadableStream).  
Om ni använder AI SDK-inspirerat stream-upplägg är det också förenligt (t.ex. `streamText`/SSE).

```mermaid
flowchart TD
  A[POST /api/v0/chats/stream] --> B[Route wrapper: create]
  A2[POST /api/v0/chats/[chatId]/stream] --> B2[Route wrapper: follow-up]

  B --> C[Resolve provider]
  B2 --> C

  C -->|legacy v0 provider| V[V0ProviderAdapter]
  C -->|own-engine| S[OwnEngineBuildSession]

  S --> G[Contract gate helper]
  G -->|awaiting input| R1[SSE: meta + awaiting_input + done]
  G -->|ok| P[Plan-mode helper?]
  P -->|plan-mode| R2[SSE or JSON: meta + plan payload]
  P -->|generate| STR[Create generation stream]

  STR --> F[Finalize: prepare in-memory]
  F --> TX[Persist atomiskt: assistant + version]
  TX --> OUT[SSE events: done (versionId), previewUrl]
  OUT --> SB[Optional: sandbox-ready event]
```

Mermaid-flödessyntax och hur den renderas i Markdown stöds brett (t.ex. på GitHub).

## Patchpaket

Det här är “runnable pack”-delen: en fil att lägga i repot och köra, plus de TypeScript-filer den genererar.

### scripts/own-engine-remediation.mjs

Denna fil är designad för att vara **körbar i Node** (utan TypeScript-runtime). Den hittar relevanta filer (heuristik), gör säkra mekaniska ändringar och scaffolder nya TS-moduler och tests. Den skriver inte över legacy utan lägger wrappers och feature flags för inkrementell migrering.

> **Implementationsnotis:** Om du mater ut skriptet till en riktig `scripts/own-engine-remediation.mjs` måste alla `const TEMPLATE_*` ligga **före** `parseArgs`/`task*`-anropen (eller exporteras till separata moduler) — annars får du temporal dead zone vid körning. Nedan är ordningen som i originalet (referens).

```js
// File: scripts/own-engine-remediation.mjs
// Node >= 18 rekommenderas (Next.js & streaming; tester kan köras via Node/Vitest)
// Kör: node scripts/own-engine-remediation.mjs --help

import fs from 'node:fs';
import path from 'node:path';
import crypto from 'node:crypto';

const DEFAULT_IGNORES = new Set([
  'node_modules',
  '.git',
  '.next',
  'dist',
  'build',
  'out',
  '.turbo',
  '.vercel',
]);

function parseArgs(argv) {
  const args = new Set(argv.slice(2));
  const get = (name) => args.has(name);
  const val = (prefix) => {
    const match = argv.find((a) => a.startsWith(prefix + '='));
    return match ? match.slice(prefix.length + 1) : null;
  };

  return {
    help: get('--help') || get('-h'),
    apply: get('--apply'),
    dryRun: get('--dry-run') || !get('--apply'),
    verbose: get('--verbose'),

    // Task toggles
    all: get('--all'),
    scaffold: get('--scaffold'),
    renameFallback: get('--rename-fallback'),
    removeStreamResolve: get('--remove-stream-resolve'),
    patchPlanMode: get('--patch-plan-mode'),
    wrapRoutes: get('--wrap-routes'),
    addTests: get('--add-tests'),
    patchPackageJson: get('--patch-package-json'),

    // Optional overrides
    srcRoot: val('--src-root'), // allows forcing "src" or ""
    routeCreate: val('--route-create'), // override path to create route.ts
    routeFollowup: val('--route-followup'), // override path to follow-up route.ts
  };
}

function sha1(s) {
  return crypto.createHash('sha1').update(s).digest('hex').slice(0, 10);
}

function exists(p) {
  try {
    fs.accessSync(p);
    return true;
  } catch {
    return false;
  }
}

function readFile(p) {
  return fs.readFileSync(p, 'utf8');
}

function writeFile(p, content) {
  fs.mkdirSync(path.dirname(p), { recursive: true });
  fs.writeFileSync(p, content, 'utf8');
}

function writeFileIfChanged(p, content, { dryRun, verbose }) {
  const prev = exists(p) ? readFile(p) : null;
  if (prev === content) {
    if (verbose) console.log(`= unchanged ${p}`);
    return { changed: false };
  }
  if (dryRun) {
    console.log(`~ would write ${p} (diff hash ${sha1((prev ?? '') + '->' + content)})`);
    return { changed: true, dryRun: true };
  }
  writeFile(p, content);
  console.log(`+ wrote ${p}`);
  return { changed: true };
}

function moveFile(from, to, { dryRun }) {
  if (dryRun) {
    console.log(`~ would move ${from} -> ${to}`);
    return;
  }
  fs.mkdirSync(path.dirname(to), { recursive: true });
  fs.renameSync(from, to);
  console.log(`> moved ${from} -> ${to}`);
}

function walk(rootDir, { ignores = DEFAULT_IGNORES } = {}) {
  const out = [];
  const stack = [rootDir];

  while (stack.length) {
    const dir = stack.pop();
    const entries = fs.readdirSync(dir, { withFileTypes: true });

    for (const e of entries) {
      if (ignores.has(e.name)) continue;
      const p = path.join(dir, e.name);
      if (e.isDirectory()) stack.push(p);
      else out.push(p);
    }
  }
  return out;
}

function findBySuffix(files, suffix) {
  const normSuffix = suffix.split('/').join(path.sep);
  return files.filter((f) => f.endsWith(normSuffix));
}

function findByBasename(files, basename) {
  return files.filter((f) => path.basename(f) === basename);
}

function findByContentRegex(files, regex, { ext = new Set(['.ts', '.tsx', '.js', '.mjs', '.cjs']) } = {}) {
  const hits = [];
  for (const f of files) {
    const e = path.extname(f);
    if (!ext.has(e)) continue;
    let text;
    try {
      text = readFile(f);
    } catch {
      continue;
    }
    if (regex.test(text)) hits.push(f);
  }
  return hits;
}

function pickBest(paths, preferredSubstrings = []) {
  if (paths.length <= 1) return paths[0] ?? null;
  for (const sub of preferredSubstrings) {
    const found = paths.find((p) => p.includes(sub));
    if (found) return found;
  }
  return paths[0];
}

function detectSrcRoot(repoRoot, argOverride) {
  if (argOverride !== null) return argOverride;
  if (exists(path.join(repoRoot, 'src'))) return 'src';
  return '';
}

function taskRenameFallback(repoRoot, files, args) {
  const enabled = args.all || args.renameFallback;
  if (!enabled) return;

  const fallbackCandidates = findBySuffix(files, `${path.sep}lib${path.sep}gen${path.sep}fallback.ts`)
    .concat(findByBasename(files, 'fallback.ts'));

  const fallbackPath = pickBest(fallbackCandidates, [
    `${path.sep}src${path.sep}lib${path.sep}gen${path.sep}fallback.ts`,
    `${path.sep}lib${path.sep}gen${path.sep}fallback.ts`,
  ]);

  if (!fallbackPath) {
    console.log('! fallback.ts not found (UNSPECIFIED). Skipping renameFallback.');
    return;
  }

  const dir = path.dirname(fallbackPath);
  const newPath = path.join(dir, 'generation-pipeline.ts');

  if (!exists(newPath)) {
    const content = readFile(fallbackPath);
    if (args.dryRun) console.log(`~ would create ${newPath} from existing fallback.ts content`);
    else writeFile(newPath, content), console.log(`+ created ${newPath}`);

    // Replace fallback.ts with wrapper (migration stub)
    const hasDefaultExport = /\bexport\s+default\b/.test(content);
    const wrapper = `/**
 * @deprecated Migrerad från fallback.ts. Använd generation-pipeline.ts.
 * Denna wrapper finns för bakåtkompatibilitet under migreringen.
 */
export * from './generation-pipeline';
${hasDefaultExport ? "export { default } from './generation-pipeline';" : ''}\n`;

    writeFileIfChanged(fallbackPath, wrapper, args);
  } else {
    // Ensure fallback.ts is wrapper (optional)
    const existing = readFile(fallbackPath);
    if (!existing.includes("generation-pipeline")) {
      console.log(`! ${newPath} exists men ${fallbackPath} verkar inte vara wrapper. Lämnar orört.`);
    }
  }
}

function taskRemoveStreamResolve(repoRoot, files, args) {
  const enabled = args.all || args.removeStreamResolve;
  if (!enabled) return;

  const CONSTS = ['STREAM_RESOLVE_MAX_ATTEMPTS', 'STREAM_RESOLVE_DELAY_MS'];

  for (const name of CONSTS) {
    const matches = findByContentRegex(files, new RegExp(`\\b${name}\\b`));
    if (matches.length === 0) continue;

    // Count occurrences across files
    let total = 0;
    const occurrences = [];
    for (const f of matches) {
      const text = readFile(f);
      const count = (text.match(new RegExp(`\\b${name}\\b`, 'g')) || []).length;
      total += count;
      occurrences.push({ f, count });
    }

    // If used beyond definition, skip automated removal
    if (total > matches.length) {
      console.log(`! ${name} appears ${total} times across ${matches.length} files. Not auto-removing.`);
      continue;
    }

    // Try removing lines with const definition
    for (const { f } of occurrences) {
      const text = readFile(f);
      const lines = text.split('\n');
      const filtered = lines.filter((ln) => !new RegExp(`\\b(const|let|var)\\s+${name}\\b`).test(ln));
      if (filtered.length === lines.length) continue;

      writeFileIfChanged(f, filtered.join('\n'), args);
      console.log(`- removed unused const ${name} from ${f}`);
    }
  }
}

function taskPatchPlanMode(repoRoot, files, args) {
  const enabled = args.all || args.patchPlanMode;
  if (!enabled) return;

  const hits = findByContentRegex(files, /\bcreateOwnEnginePlanModeResponse\b/);
  if (hits.length === 0) {
    console.log('! createOwnEnginePlanModeResponse not found (UNSPECIFIED). Skipping patchPlanMode.');
    return;
  }

  const target = pickBest(hits, [
    `${path.sep}own-engine`,
    `${path.sep}lib${path.sep}gen`,
    `${path.sep}lib${path.sep}`,
  ]);

  const text = readFile(target);

  // Heuristic: inside function, replace resolvePhaseModel(... "planner").modelId with modelId (use the param).
  // This is a best-effort fix to remove "unused param" and align meta with chosen modelId.
  const patched = text.replace(
    /resolvePhaseModel\(([^)]*?)"planner"([^)]*?)\)\.modelId/g,
    'modelId'
  );

  if (patched === text) {
    console.log(`! No planner resolvePhaseModel(...).modelId pattern found in ${target}. Leaving file unchanged.`);
    return;
  }

  writeFileIfChanged(target, patched, args);
  console.log(`~ patched createOwnEnginePlanModeResponse in ${target} (heuristic replace).`);
}

function taskScaffoldFiles(repoRoot, args) {
  const enabled = args.all || args.scaffold;
  if (!enabled) return;

  const srcRoot = detectSrcRoot(repoRoot, args.srcRoot);
  const base = srcRoot ? path.join(repoRoot, srcRoot) : repoRoot;

  const sessionPath = path.join(base, 'lib', 'own-engine', 'session', 'own-engine-build-session.ts');
  const helpersPath = path.join(base, 'lib', 'own-engine', 'session', 'own-engine-route-helpers.ts');
  const finalizePath = path.join(base, 'lib', 'own-engine', 'finalize', 'finalize-transactional.ts');
  const ssePath = path.join(base, 'lib', 'own-engine', 'stream', 'builder-sse.ts');

  writeFileIfChanged(sessionPath, TEMPLATE_OWN_ENGINE_BUILD_SESSION, args);
  writeFileIfChanged(helpersPath, TEMPLATE_OWN_ENGINE_ROUTE_HELPERS, args);
  writeFileIfChanged(finalizePath, TEMPLATE_FINALIZE_TRANSACTIONAL, args);
  writeFileIfChanged(ssePath, TEMPLATE_BUILDER_SSE, args);
}

function taskAddTests(repoRoot, args) {
  const enabled = args.all || args.addTests;
  if (!enabled) return;

  // Put tests in ./test to be framework-agnostic.
  const testRoot = path.join(repoRoot, 'test', 'own-engine');
  writeFileIfChanged(
    path.join(testRoot, 'stream.golden.test.ts'),
    TEMPLATE_TEST_STREAM_GOLDEN,
    args
  );
  writeFileIfChanged(
    path.join(testRoot, 'finalize.transactional.test.ts'),
    TEMPLATE_TEST_FINALIZE_TRANSACTIONAL,
    args
  );

  // Optional vitest config if they choose vitest
  const vitestConfig = path.join(repoRoot, 'vitest.config.ts');
  if (!exists(vitestConfig)) {
    writeFileIfChanged(vitestConfig, TEMPLATE_VITEST_CONFIG, args);
  }
}

function taskPatchPackageJson(repoRoot, args) {
  const enabled = args.all || args.patchPackageJson;
  if (!enabled) return;

  const pkgPath = path.join(repoRoot, 'package.json');
  if (!exists(pkgPath)) {
    console.log('! package.json not found. Skipping patchPackageJson.');
    return;
  }

  const pkg = JSON.parse(readFile(pkgPath));
  pkg.scripts = pkg.scripts || {};

  // Add "safe" scripts that won't break if tools are missing; agent may need to install devDeps.
  pkg.scripts['typecheck'] = pkg.scripts['typecheck'] || 'tsc --noEmit';
  pkg.scripts['test'] = pkg.scripts['test'] || 'vitest run';
  pkg.scripts['test:watch'] = pkg.scripts['test:watch'] || 'vitest';
  pkg.scripts['test:coverage'] = pkg.scripts['test:coverage'] || 'vitest run --coverage';

  // Do not assume eslint/prettier; add only if already present in deps
  const deps = { ...(pkg.dependencies || {}), ...(pkg.devDependencies || {}) };
  if (deps.eslint) pkg.scripts['lint'] = pkg.scripts['lint'] || 'eslint .';

  // Add vitest if missing (opt-in, minimal external assumptions beyond npm/pnpm install)
  pkg.devDependencies = pkg.devDependencies || {};
  if (!pkg.devDependencies.vitest) pkg.devDependencies.vitest = '^2.1.0';
  if (!pkg.devDependencies['@vitest/coverage-v8']) pkg.devDependencies['@vitest/coverage-v8'] = '^2.1.0';
  if (!pkg.devDependencies.typescript) pkg.devDependencies.typescript = pkg.devDependencies.typescript || '^5.6.0';

  const out = JSON.stringify(pkg, null, 2) + '\n';
  writeFileIfChanged(pkgPath, out, args);
}

function taskWrapRoutes(repoRoot, files, args) {
  const enabled = args.all || args.wrapRoutes;
  if (!enabled) return;

  // Heuristics for default locations; you can override via --route-create=... --route-followup=...
  const createCandidates = args.routeCreate
    ? [path.resolve(repoRoot, args.routeCreate)]
    : findBySuffix(files, `${path.sep}app${path.sep}api${path.sep}v0${path.sep}chats${path.sep}stream${path.sep}route.ts`);

  const followCandidates = args.routeFollowup
    ? [path.resolve(repoRoot, args.routeFollowup)]
    : findBySuffix(files, `${path.sep}app${path.sep}api${path.sep}v0${path.sep}chats${path.sep}[chatId]${path.sep}stream${path.sep}route.ts`);

  const createPath = pickBest(createCandidates, [
    `${path.sep}src${path.sep}app${path.sep}api${path.sep}v0${path.sep}chats${path.sep}stream${path.sep}route.ts`,
  ]);
  const followPath = pickBest(followCandidates, [
    `${path.sep}src${path.sep}app${path.sep}api${path.sep}v0${path.sep}chats${path.sep}[chatId]${path.sep}stream${path.sep}route.ts`,
  ]);

  for (const p of [createPath, followPath]) {
    if (!p) continue;
    if (!exists(p)) continue;

    const legacyPath = p.replace(/route\.ts$/, 'route.legacy.ts');
    if (!exists(legacyPath)) {
      moveFile(p, legacyPath, args);
    } else {
      console.log(`= legacy already exists ${legacyPath}, leaving original route.ts untouched`);
      continue;
    }

    const isFollowup = p.includes('[chatId]');
    const wrapper = isFollowup ? TEMPLATE_ROUTE_WRAPPER_FOLLOWUP : TEMPLATE_ROUTE_WRAPPER_CREATE;
    writeFileIfChanged(p, wrapper, args);

    console.log(`~ wrapped route ${p}. Default behavior stays legacy unless OWN_ENGINE_SESSION_V2=1`);
  }
}

const TEMPLATE_OWN_ENGINE_BUILD_SESSION = `/* eslint-disable @typescript-eslint/no-explicit-any */
/**
 * File: src/lib/own-engine/session/own-engine-build-session.ts
 *
 * UNSPECIFIED: Den här filen är en “refactor target”. Cursor-agenten ska
 * flytta delad create/follow-up route-logik hit stegvis.

export type OwnEngineRouteKind = 'create' | 'followup';

export type BuildProviderKind = 'own-engine' | 'v0-provider';

export interface OwnEngineBuildSessionArgs {
  kind: OwnEngineRouteKind;
  request: Request;
  params?: Record<string, string | string[] | undefined>;
  featureFlags?: {
    /**
     * Feature flag för att slå på nya session-flödet (routes kan defaulta till legacy).
     */
    enableSessionV2?: boolean;
    /**
     * Om true: finalize ska försöka göra atomisk persist (transaction/pending).
     */
    enableTransactionalFinalize?: boolean;
  };

export interface OwnEngineBuildSessionDeps {
  /**
   * Provider-resolver: välj mellan own-engine och legacy v0 provider.
   * Behöver vara deterministisk och testbar.
   */
  resolveProvider(args: {
    kind: OwnEngineRouteKind;
    request: Request;
    params?: OwnEngineBuildSessionArgs['params'];
  }): Promise<BuildProviderKind> | BuildProviderKind;

  /**
   * Legacy path: om request ska gå via extern v0 provider.
   * UNSPECIFIED: wire to repo.
   */
  handleViaV0Provider(args: OwnEngineBuildSessionArgs): Promise<Response>;

  /**
   * Bygg ett “session context” från request + params.
   * Denna funktion ska samla och normalisera:
   * - auth/user
   * - chatId (followup)
   * - messages / attachments
   * - model tier/model id
   * - scaffold info
   * - contract inference och om clarification krävs
   * - plan mode flag
   *
   * UNSPECIFIED: wire to repo’s existing request parsing + orchestration code.
   */
  createSessionContext(args: OwnEngineBuildSessionArgs): Promise<OwnEngineSessionContext>;

  /**
   * Om kontraktet kräver fler uppgifter: bygg ett awaiting-input response,
   * helst via stream-kontraktet.
   */
  buildContractClarificationResponse(ctx: OwnEngineSessionContext): Promise<Response>;

  /**
   * Om plan-mode: bygg en plan-mode response (kan vara stream eller JSON).
   */
  buildPlanModeResponse(ctx: OwnEngineSessionContext): Promise<Response>;

  /**
   * Skapa generation-stream (SSE eller annat egen-kontrakt).
   * Det här ska vara den “enhetliga” entry pointen för own-engine generation.
   * UNSPECIFIED: wire to repo’s generation pipeline + stream adapter.
   */
  createGenerationResponse(ctx: OwnEngineSessionContext): Promise<Response>;
}

export interface OwnEngineSessionContext {
  // Minimalt behövs:
  kind: OwnEngineRouteKind;
  // ...allt annat är repo-specifikt:
  [k: string]: any;

  // Standardiserade flags:
  requiresContractClarification?: boolean;
  isPlanMode?: boolean;
}

export class OwnEngineBuildSession {
  private readonly args: OwnEngineBuildSessionArgs;
  private readonly deps: OwnEngineBuildSessionDeps;

  constructor(args: OwnEngineBuildSessionArgs, deps: OwnEngineBuildSessionDeps) {
    this.args = args;
    this.deps = deps;
  }

  async handle(): Promise<Response> {
    const provider = await this.deps.resolveProvider({
      kind: this.args.kind,
      request: this.args.request,
      params: this.args.params,
    });

    if (provider === 'v0-provider') {
      return this.deps.handleViaV0Provider(this.args);
    }

    const ctx = await this.deps.createSessionContext(this.args);

    if (ctx.requiresContractClarification) {
      return this.deps.buildContractClarificationResponse(ctx);
    }

    if (ctx.isPlanMode) {
      return this.deps.buildPlanModeResponse(ctx);
    }

    return this.deps.createGenerationResponse(ctx);
  }
}

const TEMPLATE_FINALIZE_TRANSACTIONAL = `/* eslint-disable @typescript-eslint/no-explicit-any */
/**
 * File: src/lib/own-engine/finalize/finalize-transactional.ts
 *
 * Mål:
 *  - undvika orphan assistant messages
 *  - göra finalize tvåfasig:

 *      2) persist (assistant + version) atomiskt / i transaktion
 *
 * UNSPECIFIED:
 *  - DB/ORM (Prisma/Drizzle/etc)
 *  - Vad “assistant message” och “version” exakt heter i er domän
 */

export interface PreparedFinalizeArtifacts {
  // Text som ska sparas i assistant message
  assistantText: string;

  // Projektfiler/diff efter parse/merge
  files: Array<{ path: string; content: string }>;

  // Preflight-resultat
  preflight: {
    ok: boolean;
    warnings?: string[];
    errors?: string[];
    // repo-specifikt
    [k: string]: any;
  };

  // Extra metadata (telemetry etc)
  meta?: Record<string, any>;
}

export interface PersistResult {
  assistantMessageId: string;
  versionId: string;
}

export interface FinalizeTransactionalDeps<TTx = any> {
  /**
   * Skapa en transaktion om DB stödjer det.
   * Om ni inte har transactions: returnera callback-result direkt,
   * men då bör ni aktivera “pending assistant” strategi.
   */
  withTransaction<R>(fn: (tx: TTx) => Promise<R>): Promise<R>;

  /**
   * Atomisk persist av version + assistant message.
   * Rekommendation: gör båda writes i samma tx.
   */
  persistAssistantAndVersion(tx: TTx, args: {
    ctx: any; // repo-specifikt finalize context (chatId, userId, etc)
    prepared: PreparedFinalizeArtifacts;
  }): Promise<PersistResult>;

  /**
   * Optional: pending-strategi om ni inte kan göra riktiga transaktioner.
   * Implementera om ni vill kunna “markera pending” och sedan commit.
   */
  createPendingAssistantMessage?(
    tx: TTx,
    args: { ctx: any; assistantText: string; meta?: Record<string, any> }
  ): Promise<{ assistantMessageId: string }>;

  commitPendingAssistantMessage?(
    tx: TTx,
    args: { assistantMessageId: string; versionId: string }
  ): Promise<void>;
}

/**
 * Tvåfasig finalize:
 * 1) prepare i minnet (ingen DB)
 * 2) persist atomiskt
 */
export async function finalizeTransactional<TTx = any>(args: {
  ctx: any;
  prepare(): Promise<PreparedFinalizeArtifacts>;
  deps: FinalizeTransactionalDeps<TTx>;
}): Promise<PersistResult> {
  const prepared = await args.prepare();

  // Tidig fail om preflight inte ok (policy kan vara repo-specifik)
  if (!prepared.preflight.ok) {
    // Här kan ni välja att ändå spara en “failed” version, eller bara returnera error.
    throw new Error(
      'Preflight failed (policy UNSPECIFIED). Implementera repo-specifik hantering.'
    );
  }

  // Atomisk persist
  return args.deps.withTransaction(async (tx) => {
    // Om pending-strategi används:
    // 1) skapa pending assistant
    // 2) skapa version
    // 3) commit pending
    if (args.deps.createPendingAssistantMessage && args.deps.commitPendingAssistantMessage) {
      const pending = await args.deps.createPendingAssistantMessage(tx, {
        ctx: args.ctx,
        assistantText: prepared.assistantText,
        meta: prepared.meta,
      });

      const pv = await args.deps.persistAssistantAndVersion(tx, {
        ctx: args.ctx,
        prepared,
      });

      await args.deps.commitPendingAssistantMessage(tx, {
        assistantMessageId: pending.assistantMessageId,
        versionId: pv.versionId,
      });

      return pv;
    }

    // Default: en dependency gör båda writes i en tx
    return args.deps.persistAssistantAndVersion(tx, {
      ctx: args.ctx,
      prepared,
    });
  });
}
`;

const TEMPLATE_BUILDER_SSE = `/* eslint-disable @typescript-eslint/no-explicit-any */
/**
 * File: src/lib/own-engine/stream/builder-sse.ts
 *
 * Syfte:
 *  - tajta stream payload-typer (minst inom own-engine)
 *  - erbjuda encoder/decoder för SSE för golden tests
 *
 * UNSPECIFIED:
 *  - Exakta event-namn och payload-shapes i repo’s builder-stream-contract.ts.
 *    Cursor-agenten ska mappa dessa typer till befintligt kontrakt.
 */

export type BuilderEvent =
  | { type: 'meta'; payload: Record<string, any> }
  | { type: 'awaiting_input'; payload: { prompt: string; [k: string]: any } }
  | { type: 'text_delta'; payload: { delta: string } }
  | { type: 'tool_call'; payload: Record<string, any> }
  | { type: 'error'; payload: { message: string; code?: string; [k: string]: any } }
  | { type: 'done'; payload: { versionId?: string | null; previewUrl?: string | null; [k: string]: any } }
  | { type: 'sandbox_ready'; payload: { url: string; [k: string]: any } };

const encoder = new TextEncoder();

export function encodeSseEvent(evt: BuilderEvent): Uint8Array {
  // SSE: event: <name> \\n data: <json> \\n\\n
  const body = JSON.stringify(evt.payload);
  const s = \`event: \${evt.type}\\ndata: \${body}\\n\\n\`;
  return encoder.encode(s);
}

export function createSseStreamFromEvents(events: AsyncIterable<BuilderEvent>): ReadableStream<Uint8Array> {
  return new ReadableStream<Uint8Array>({
    async start(controller) {
      try {
        for await (const evt of events) controller.enqueue(encodeSseEvent(evt));
        controller.close();
      } catch (err: any) {
        const msg = err?.message ?? String(err);
        controller.enqueue(encodeSseEvent({ type: 'error', payload: { message: msg } }));
        controller.close();
      }
    },
  });
}

/**
 * Test-helper: konsumera ReadableStream till string.
 */
export async function consumeStreamToString(stream: ReadableStream<Uint8Array>): Promise<string> {
  const reader = stream.getReader();
  const chunks = [];
  while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    chunks.push(value);
  }
  const out = new Uint8Array(chunks.reduce((n, c) => n + c.length, 0));
  let offset = 0;
  for (const c of chunks) {
    out.set(c, offset);
    offset += c.length;
  }
  return new TextDecoder().decode(out);
}
`;

### Nya TypeScript-moduler och tester

Patch-runnern skriver filerna ovan (session/helpers/finalize/sse/tests). De är avsiktligt konstruerade så att:

- de **kompilerar** även innan full wiring är klar (med `any` + “UNSPECIFIED”),
- agenten kan flytta logik från `route.legacy.ts` till `OwnEngineBuildSession` stegvis,
- tests ger “golden” bas för stream encoding och finalize-atomicitet.

För format och giltig Next.js route handler-signatur hänvisas till Route Handler-dokumentationen.  
För streaming (SSE/streaming responses) finns etablerade mönster; om ni sitter på AI SDK-liknande protokoll kan detta mappas direkt.

## Verifiering och agentens körschema

Det här är ett steg-för-steg-schema som Cursor-agenten kan köra (utan att fråga dig om något), med inkrementell riskkontroll och feature flags.

### Exekveringsordning

1) **Kör discovery + dry-run**
- `node scripts/own-engine-remediation.mjs --dry-run --all`

2) **Applicera säkra förändringar först**
- Rename + wrappers + constants + plan-mode patch:
  - `node scripts/own-engine-remediation.mjs --apply --rename-fallback --remove-stream-resolve --patch-plan-mode`

3) **Scaffolda session/finalize/stream + tester**
- `node scripts/own-engine-remediation.mjs --apply --scaffold --add-tests`

4) **Valfritt: patcha package.json för snabb lokalkörning**
- `node scripts/own-engine-remediation.mjs --apply --patch-package-json`

5) **Installera deps och kör checks lokalt**
- `pnpm install` (eller `npm install` / `yarn`)  
- `pnpm typecheck` (kräver `tsc`)  
- `pnpm test` (om vitest patchades in)

Node’s test runner är ett alternativ om ni vill undvika Vitest, men TypeScript kräver då extra loader/kompilering. Därför är Vitest här ett pragmatiskt default-val för en TS-kodbas. Node dokumenterar test runner och täckning separat.

6) **Wrap routes (riskkontrollerat, feature-flag)**
- Kör först dry-run:
  - `node scripts/own-engine-remediation.mjs --dry-run --wrap-routes`
- Applicera sedan:
  - `node scripts/own-engine-remediation.mjs --apply --wrap-routes`

7) **Aktivera nya sessionen i dev (utan att bryta prod)**
- Sätt `OWN_ENGINE_SESSION_V2=1` lokalt och verifiera att endpoints svarar.
- Om något fallerar: ta bort flaggan → ni faller tillbaka till `route.legacy.ts`.

### Checklist för “klar” refaktor

När agenten har flyttat logik från legacy till session är målet att:

- `route.ts` i båda endpoints enbart:
  - parse ctx,
  - skapar deps/wiring,
  - `return new OwnEngineBuildSession(...).handle()`,
  - samt eventuellt väljer v0-provider via resolver.

- `OwnEngineBuildSession` är ensam plats för:
  - contract gate,
  - plan mode,
  - start av generation stream,
  - finalize hook.

- Finalize är tvåfasig:
  - `prepare()` gör parse/merge/preflight i minne,
  - `persist...` sker i transaction/pending och kan inte lämna orphan messages.

### Snabba CI-liknande kommandon

Den minsta “lokal CI” (ingen extern infra) blir:

- `pnpm typecheck`
- `pnpm test`
- (valfritt) `pnpm lint` om eslint finns

Detta upplägg är i linje med TypeScript-kompilering och test runner-koncept.
