================================================================================
                    CORE USER FLOW (SIMPLIFIED)
================================================================================

Step 1: CHOOSE PATH
-------------------
User lands on platform and selects:
  A) "Use a template" → Shows template gallery
  B) "Describe your site" → Free-form prompt input
  C) "Upload Figma" → Figma file upload (optional)

Step 2: INITIAL GENERATION
--------------------------
System calls v0.chats.create() with:
  - User's description OR selected template
  - Returns: generated code + preview URL

Step 3: PREVIEW & ITERATE
-------------------------
User sees:
  - Live preview (iframe)
  - Generated code (optional view)
  - Chat input for refinements

User can say: "Make the header sticky" → v0 updates code

Step 4: FINALIZE
----------------
When satisfied, user chooses:
  - Download code (ZIP)
  - Push to GitHub
  - Deploy to Vercel
  - Continue editing

================================================================================
                    TECHNICAL REQUIREMENTS
================================================================================

Frontend Stack
--------------
- Next.js 14+ (App Router)
- React 18+
- TypeScript
- Tailwind CSS
- ShadCN UI components

Backend/API
-----------
- v0 Platform API (@v0-sdk)
- API key stored in environment variables
- Rate limiting awareness

Key API Calls
-------------
1. v0.chats.create({ message: '...' })     → Create new generation
2. v0.chats.sendMessage({ chatId, ... })   → Continue conversation
3. v0.chats.getById(chatId)                → Fetch chat history
4. chat.files                              → Get generated files
5. chat.demo                               → Get preview URL

================================================================================
                    RISKS & MITIGATIONS
================================================================================

Risk: API rate limiting
Mitigation: Implement request queuing, show "generating..." states

Risk: Generated code quality varies
Mitigation: Provide example prompts, template starting points

Risk: Preview iframe blocked
Mitigation: Fallback to code-only view with syntax highlighting

Risk: User expects too much
Mitigation: Clear onboarding explaining AI limitations
