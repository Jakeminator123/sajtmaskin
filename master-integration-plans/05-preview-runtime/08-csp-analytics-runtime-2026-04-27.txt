## Local Preview UX To Preserve
- **Parent** app: nonce `x-csp-nonce` → `RootLayout` + `json-ld` script tags — samma mönster i SEO pages (`render-seo-page`).
- SEO paths med extra `script-src-elem` `unsafe-inline` där policy säger så; `/api/preview-render` särskild policy.
- CORS: `ALLOWED_ORIGINS` + `VERCEL_URL` i `proxy` — preview och API anrop ska svara med rätt origin när lokal+preview_port blandat.
- **I embed:** om preview-**VM**-URL sätter `X-Frame-Options` / `CSP` som blockerar inbäddning, är det **preview origin**-problem, inte enbart parent `next.config`.

## Failure Modes
- `CSP` enforce i prod: inline script utan nonce/whitelist bryter layout.
- Miss `proxy` i deployment: CORS/edge auth borta (verifiera Next 16 wiring — kommentar i källan `proxy` ersätter middleware).
- Vercel insights dubbel ladda om `VERCEL_ENV` satt i fel miljö.

## Verification Matrix
- Dev: ladda `/`, öppna konsol — inga brutna CSP (report-only: kolla `csp-report` endpoint logg).
- Prod: `CSP_ENFORCE` staging — hard refresh alla marketing routes + builder entry.
- JSON-LD: `view-source` se `nonce` satt, ogiltig JSON 0.
- Iframe: om preview laddar men blank — kolla nätverk på **barn-**origin CSP.

## Do Not Break
- Nonce pipeline från edge till layout (ingen ad hoc `eval` nya ställen).
- `COOP/COEP` endast där tänkt (`/project/:path*` WebContainer) — blanda ej med preview-iframe.
- CORS-allow inte vidga till vilket skämt-ursprung; håll `getAppBaseUrl()` sann.
- Lokal: `vercel-insights` ska förbli tyst mot localhost (beteende bevarat).
