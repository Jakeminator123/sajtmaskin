1. BEROENDEKARTAN – VAD ÄGS AV VEM?
================================================================================

1.1 V0 (Vercels AI) – Det kritiska beroendet
---------------------------------------------
V0 är hjärtat i produkten. All sidgenerering går genom V0:s Platform API:
- v0-sdk + V0 Platform API → skapar själva sajterna
- V0 Model API (api.v0.dev/v1) → OpenAI-kompatibel chat för prompt assist
- V0 webhooks → chat.created, message.finished, deployment.ready
- Modeller: v0-max-fast, v0-1.5-lg, v0-1.5-md, v0-gpt-5

Sajtmaskins egen intelligens runt V0: Prompt orchestration, buildIntent,
promptAssist, kvalitetsgating – custom och värdefullt, men byggt runt V0:s motor.

1.2 Vercel Platform – Infrastruktur
------------------------------------
- Deployment API → Publicerar kundsajter till Vercel
- Vercel Blob → Media, AI-bilder, uppladdningar
- Vercel Sandbox → Kör kod för quality gate/preview
- Vercel Domains API → Domänhantering åt kunder
- AI Gateway → Routar till OpenAI/Claude
- Analytics + Speed Insights → Övervakning
- Serverless Functions → AWS Lambda under huven (Node.js 20.x)
- maxDuration: 300s för /api/template (kräver Pro-plan)

1.3 OpenAI – Stödfunktioner
----------------------------
- GPT-5.2 för prompt assist, audit, wizard
- GPT-5-mini för wizard/enrich, competitors, company-lookup
- GPT-5-nano för textanalys
- text-embedding-3-small för template-sökning
- Whisper (whisper-1) för rösttranskription

1.4 Övrigt
----------
- Supabase (PostgreSQL) – databas, Drizzle ORM
- Upstash Redis – rate limiting
- Stripe – betalningar/credits
- Resend – e-post
- Brave Search – företagsuppslag
- Loopia – domänhantering (valfritt)
- OpenClaw – Sajtagenten (valfritt)

2. FULLSTÄNDIG PROCESSKARTA – FRÅN BESÖK TILL PUBLICERAD SAJT
================================================================================

FAS 0 – Användarens ingång
---------------------------
Användaren landar på sajtmaskin.se (Next.js på Vercel).

5 ingångslägen:
- Fritext: ChatArea → fri beskrivning
- Wizard: PromptWizardModalV2 → flersteg (företag, bransch, ort, färger, mål)
- Kategori/Mall: TemplateGallery → branschmall eller prompt inom kategori
- Audit: SiteAuditSection → klistra in URL → AI analyserar befintlig sajt
- Kostnadsfri: Lösenordsskyddad mini-wizard

FAS 1 – Datainsamling och projektstart
----------------------------------------
1. POST /api/projects → Skapar projekt i Supabase (PostgreSQL via Drizzle)
2. POST /api/prompts → Sparar prompten med source
3. router.push("/builder?project=X&promptId=Y&buildMethod=Z&buildIntent=website")

Wizard-specifikt:
- POST /api/company-profile (sparar företagsdata)
- POST /api/wizard/enrich (AI: gpt-5-mini, per wizard-steg)
- POST /api/wizard/company-lookup (Cheerio → Brave → AI fallback)
- POST /api/wizard/competitors (AI: gpt-5-mini)

FAS 2 – Builder laddas, prompt förbättras (valfritt)
------------------------------------------------------
1. GET /api/prompts/[promptId] → Hämtar sparad prompt
2. usePromptAssist.maybeEnhanceInitialPrompt():
   - Shallow (rewrite/polish): POST /api/ai/chat
     Provider: AI Gateway ELLER V0 Model API
     Modell: openai/gpt-5.2 eller v0-1.5-md
   - Deep Brief: POST /api/ai/brief
     Provider: AI Gateway, Modell: openai/gpt-5.2
     Output: Strukturerat JSON-brief → buildV0PromptFromBrief()
3. generateDynamicInstructions() → system prompt med design-instruktioner

FAS 3 – Prompt Orchestration (server-side)
-------------------------------------------
POST /api/v0/chats/stream → orchestratePromptMessage():
1. Normalisera meddelande
2. Detektera prompt-typ (audit/wizard/freeform/template/followup)
3. Resolve soft target (3200-6500 tecken)
4. Analysera komplexitet (rader, punkter, rubriker, URL:er, nyckelord)
5. Välj strategi:
   - direct: inom budget → skicka som den är
   - summarize: för lång → komprimera
   - phase_plan_build_polish: mycket komplex → dela i faser

FAS 4 – V0 kodgenerering (kärnan)
-----------------------------------
POST https://api.v0.dev/v1/chats
Headers: Authorization: Bearer V0_API_KEY
Body: {
  message: optimizedMessage,
  system: systemPrompt,
  chatPrivacy: "private",
  modelConfiguration: {
    modelId: "v0-max-fast" | "v0-1.5-md" | "v0-1.5-lg" | "v0-gpt-5",
    imageGenerations: true/false,
    thinking: true/false
  },
  responseMode: "experimental_stream",
  attachments: [{ url: "..." }],
  designSystemId: "..."
}

Maskin: V0:s egna servrar (Vercel-ägd, troligen GPU-baserade)
Output: SSE-ström med genererad React/Next.js-kod

FAS 5 – Streaming till klienten
---------------------------------
Stream-route (Vercel Serverless):
1. Läs V0:s SSE-ström via TextDecoder
2. Parsa event:/data:-rader
3. Extrahera via v0Stream: demoUrl, versionId, content, thinking, uiParts
4. Vidaresänd som egna SSE-events till klienten
5. Vid done: upserta version i Supabase

Klient (handleSseStream):
- chatId → setChatId
- content → visa kodinnehåll
- done → setCurrentDemoUrl, runPostGenerationChecks

PreviewPanel:
- iframe src = demoUrl (vusercontent.net) – hostas av V0

FAS 6 – Kvalitetskontroll (automatisk)
----------------------------------------
Klient-side:
- no_file_changes, missing_preview_url, stream_anomaly
- validate-images: POST /api/v0/chats/.../validate

Server-side (icke-blockerande):
POST /api/v0/chats/[chatId]/quality-gate
- @vercel/sandbox (Firecracker microVM)
- npm install → tsc --noEmit → next build → eslint
- Spara fel i DB

FAS 7 – Iteration
-------------------
Användaren skriver uppföljning → useSendMessage()
→ POST /api/v0/chats/[chatId]/stream
→ orchestratePromptMessage({ isFirstPrompt: false })
→ v0.chats.sendMessage → ny version genereras
→ Samma SSE-ström och preview-uppdatering

FAS 8 – Deploy
----------------
POST /api/v0/deployments:
1. Validera chat + version (DB)
2. Kreditkontroll via Stripe
3. createDeploymentRecord → Supabase
4. v0.chats.getVersion → hämta filer
5. extractTextFilesFromV0Version
6. applyPreDeployFixes (lockfiler, use client, font-weight, @utility)
7. materializeImagesInTextFiles (Vercel Blob eller external)
8. toVercelFilesFromTextFiles (base64-koda)
9. createVercelDeployment:
   POST https://api.vercel.com/v13/deployments
   Body: { name, target, files, env }
   Maskin: Vercel Build Infrastructure → Edge Network (CDN)
10. syncEnvVarsToVercelProject
11. Webhook/polling för deployment-status

FAS 9 – Domän (valfritt)
--------------------------
POST /api/domains/check → Vercel Domains API
POST /api/domains/link → Koppla domän
POST /api/domain-suggestions → AI (gpt-5.2)

I de ~6,700 raderna som utgör systemets "hjärna":

SAJTMASKINS INTELLIGENS: ~2,800 rader
--------------------------------------

  3 strategier, komplexitetsscoring, 9 viktade regler, sammanfattning, fasplanering

  Motion-profiler (44 nyckelord), visuell identitet, kvalitetsnivåer, brief→prompt
- Kategori-prompter: ~230 rader
  5 domänspecifika prompter (landing page, e-handel, dashboard, portfolio, webbplats)
- Audit-prompter: ~380 rader
  200 rader systeminstruktioner, JSON-schema, svenska regler
- Stream-intelligens: ~280 rader
  Tool-state-mappning, integration-signaler, delta-parsning
- Post-checks: ~280 rader
  Fildiffar, saknade routes, suspekt use(), design-tokens, regressionmatris
- Pre-deploy fixes: ~145 rader
  @utility-fix, "use client"-detektion, font-weight-fix, lockfile-rensning
- Wizard-logik: ~180 rader
  11 branschprofiler, designfeatures, konditionella steg
- Promptgränser: ~60 rader
  17 tunade parametrar

V0:s INTELLIGENS (osynlig, inte i kodbasen):
----------------------------------------------
- Finetunad kodmodell för Next.js/React/Tailwind/shadcn
- Multi-fil-output (10-30 sammanhängande filer)
- Design-förståelse (minimalistisk, lekfull, professionell → Tailwind)
- Integrerad AI-bildgenerering
- Konsekvent hög kvalitet

Fördelning av slutproduktens kvalitet:
- V0 kodgenerering: ~55%
- Sajtmaskins orchestration: ~35%
- OpenAI stödfunktioner: ~10%

Sajtmaskins intelligens är "multiplikatorn" – tar V0 från 6/10 till 9/10
för svenska SMBs. Men V0 är fortfarande "basmotorn".

5. UTBRYTBARA PUSSELBITAR – JUICEFACTORY + QWEN
================================================================================

5.1 LÄTT (1-2 dagar per bit, ingen funktionsförlust)
------------------------------------------------------
AI Gateway → Qwen direkt
  10 filer, byt gateway("openai/gpt-5.2") mot Qwen-endpoint
  ~100 rader ändringar totalt

Vercel Blob → MinIO/S3 på JUICEFactory
  1 wrapper-fil: blob-service.ts (291 rader)
  Byt @vercel/blob → @aws-sdk/client-s3

Redis → Egen Redis
  1 fil: src/lib/data/redis.ts
  Byt @upstash/redis → ioredis

PostgreSQL → Egen PG
  1 fil: src/lib/db/client.ts – bara connection string
  Drizzle ORM = noll kodändringar utöver env-var

5.2 MEDEL (1-2 veckor)
------------------------
Vercel Deploy → Docker-baserad deploy
  vercelDeploy.ts (236 rader) + vercel-client.ts (696 rader)
  Bygg Docker-image, pusha till registry, starta container
  Kundsajter på kund.juicefactory.se istället för kund.vercel.app
  ~500-800 rader ny kod

Vercel Sandbox → Docker-container
  sandbox/route.ts (251 rader), quality-gate/route.ts (185 rader)
  Docker-container med Node.js, timeout + cleanup
  ~300-400 rader ny kod

OpenAI → Qwen för stödfunktioner
  9 filer med direkt openai-import
  Undantag: Whisper (rösttranskription) – behåll eller self-host
  Undantag: Embeddings – kräver re-embedding av alla templates

5.3 SVÅRT/STRATEGISKT (månader)
---------------------------------
V0 kodgenerering → Egen kodgenerator med Qwen Coder 72B
  Fas A (månad 1-2): Egen generation backend
    Qwen Coder 72B via vLLM på GPU-server
    System prompt med Next.js + Tailwind + shadcn
    Output: JSON med { files: [{ name, content }] }
    Egen SSE-stream, preview via Docker
  Fas B (månad 2-3): Kvalitetsiteration
    A/B-test V0 vs Qwen på riktiga kundprompts
    Finetuning på bästa V0-outputs
  Fas C (månad 3-6): Bildgenerering
    Stable Diffusion / FLUX på GPU
    Integration med kodgenereringen

JUICEFactory kan totalt ersätta:
  Dag 1:       Infra (DB, Redis, Blob)         → 100%
  Vecka 1-2:   Deploy + Sandbox                → 100%
  Vecka 2-4:   AI-inference (Qwen)             → ~85%
  Månad 1-3:   V0 kodgenerering (Qwen Coder)   → 60-80%
  Månad 3-6:   Finetuning + bildgen            → 90%+

Kvar hos tredje part:
  - Stripe (betalningar)
  - Domänregistrering (Loopia/Vercel)
  - E-post (Resend)

================================================================================
7. STRATEGISKA REKOMMENDATIONER
================================================================================

1. ÖKA MARGINALEN
   Flytta infra till JUICEFactory. Gå från ~20% till ~50-60% marginal.

2. FÖRDJUPA NISCHEN
   Fler branscher, lokala integrationer (Google My Business, Bokadirekt).
   Ingen global aktör slår er i "svensk frisör vill ha hemsida".

3. ÄG KODGENERERINGEN PÅ SIKT
   Qwen Coder hos JUICEFactory + finetuning på bästa V0-outputs.
   3-6 månader till MVP, kvaliteten förbättras iterativt.

4. SKIPPA FULL-STACK-RACET
   Svenska SMBs behöver sällan app med databas.
   De behöver snygg sajt med kontaktformulär och Google Maps.

5. DUAL-ENGINE STRATEGI
   Behåll V0 som premium-motor.
   Bygg Qwen som billigare alternativ för enklare sajter.
   Gradvis fasa över allt eftersom kvaliteten ökar.
