C. OPENAI RESPONSES API (ersatter Chat Completions)
================================================================================
Prioritet: MEDEL
Insats: MEDEL-HOG (migration, testa noggrant)
Varde: 40-80% billigare, inbyggd websok, structured outputs, battre caching

Bakgrund:
- Responses API ar OpenAIs nya standard (mars 2025+), ersatter Chat Completions.
- Anrop: openai.responses.create({ model, input, tools: [{ type: "web_search" }] })
- Inbyggda verktyg: web_search, file_search, code_interpreter
- Idag: Bara src/app/api/text/analyze/route.ts anvander responses.create.
  Resten (ai/chat, audit, wizard/enrich) kor Chat Completions via AI SDK streamText.

OBS: web_search i Responses API ar INTE gratis - det ar OpenAIs eget websok
(kostar per sokning). Det ar darfor Brave ar ett bra alternativ for saker
som company-lookup - Brave ar billigare/gratis pa free tier.

Steg:
- [x] C1. Migrera /api/text/analyze fullt (redan halvvags, rensa manuell JSON-parse)
- [x] C2. Migrera /api/audit -> responses.create med structured output
        (eliminerar JSON-parsfel, ger garanterad schema)
- [x] C3. Testa web_search i audit-routen (for SEO-kontext, konkurrentdata)
- [x] C4. Migrera /api/wizard/enrich -> structured outputs for questions/suggestions
- [ ] C5. Utvardera kostnad: OpenAI web_search vs Brave for company-lookup
