PROBLEM
-------
v0 has 12 categories, but they are NOT equal:
- Some are STARTING POINTS (what you're building)
- Some are ENHANCEMENTS (what you add later)
- Some are FOUNDATIONS (styling that applies to everything)

Showing all 12 at once would confuse users. We need smart segmentation.

================================================================================
                    PROMPT TEMPLATES BY TIER
================================================================================

When user selects a category, these prompts are sent to v0:

TIER 1 PROMPTS (Primary - sent on initial generation):
------------------------------------------------------

LANDING_PAGE:
"Create a modern, professional landing page with:
- Hero section with headline and CTA
- Features/benefits grid
- Social proof (testimonials or logos)
- Pricing table (optional)
- FAQ section
- Contact/signup form
- Footer with links
Use [THEME] theme. Make it fully responsive."

FULL_WEBSITE:
"Create a complete multi-page website with:
- Home page with hero and key sections
- About page
- Services/products page
- Contact page with form
- Consistent header and footer across all pages
Use [THEME] theme. Include navigation between pages."

DASHBOARD:
"Create an admin dashboard with:
- Sidebar navigation with icons
- Top header with user menu and notifications
- Main content area with metric cards
- Data table with sorting/filtering
- Charts (line, bar, or pie)
- Dark theme preferred
Make it fully responsive with collapsible sidebar on mobile."

ECOMMERCE:
"Create an e-commerce storefront with:
- Header with logo, search, cart icon
- Category navigation
- Product grid with images, prices, ratings
- Filter sidebar (price, category, etc.)
- Product quick-view modal
- Footer with links and newsletter signup
Use [THEME] theme. Make it fully responsive."

BLOG:
"Create a blog website with:
- Header with logo and navigation
- Featured article hero section
- Article grid with thumbnails and excerpts
- Sidebar with categories and popular posts
- Newsletter signup section
- Footer
Use [THEME] theme. Make it fully responsive."

PORTFOLIO:
"Create a portfolio website with:
- Hero section with name and title
- About me section
- Project gallery with hover effects
- Skills/expertise section
- Testimonials (optional)
- Contact form
Use [THEME] theme. Make it fully responsive."

WEB_APP:
"Create a web application with:
- Clean, functional interface
- Main workspace area
- Toolbar or action buttons
- Settings or configuration panel
- Responsive layout
Focus on usability and functionality."


TIER 2 PROMPTS (Enhancements - sent after initial generation):
--------------------------------------------------------------

ADD_COMPONENT (with sub-type):
"Add a [COMPONENT_TYPE] section to the page. Match the existing design style.
Place it [POSITION - e.g., 'after the hero section', 'before the footer'].
Make sure it integrates seamlessly with the current layout."

Component types:
- pricing_table: "Add a pricing table with 3 tiers (Basic, Pro, Enterprise)"
- testimonials: "Add a testimonials section with customer quotes and photos"
- faq: "Add an FAQ accordion section"
- contact_form: "Add a contact form with name, email, message fields"
- newsletter: "Add a newsletter signup form"
- image_gallery: "Add an image gallery with lightbox"
- team: "Add a team section with member cards"
- stats: "Add a statistics/numbers section"
- logos: "Add a logo cloud for partners/clients"
- cta: "Add a call-to-action banner"

ADD_LOGIN_SIGNUP:
"Add authentication pages:
- Login page with email/password fields
- Sign up page with name, email, password
- Forgot password page
- Include social login buttons (Google, GitHub)
Match the existing design style."

ADD_ANIMATIONS:
"Add smooth animations to the page:
- Fade-in effects on scroll
- Subtle hover animations on cards and buttons
- Smooth transitions between states
- Loading animations where appropriate
Keep animations subtle and professional."

ADD_AI_INTERFACE:
"Add an AI chat interface:
- Chat window with message bubbles
- Input field with send button
- Typing indicator
- Dark/light mode compatible
Place it as [floating button / embedded section / full page]."


TIER 3 PROMPTS (Foundations - can be included in initial or as modification):
-----------------------------------------------------------------------------

APPLY_DESIGN_SYSTEM:
"Apply this design system to the entire project:
- Primary color: [COLOR]
- Secondary color: [COLOR]
- Background: [light/dark]
- Font: [FONT]
- Border radius: [sharp/rounded/pill]
- Shadow style: [none/subtle/pronounced]
Update all components to use these consistent styles."

APPLY_LAYOUT:
"Change the page layout to:
- [sidebar-left / sidebar-right / top-nav / centered]
- [full-width / contained / narrow]
Reorganize content to fit the new layout while keeping all sections."
