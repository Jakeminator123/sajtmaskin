## Scope
Incoming master features that must be retained and wired into the current frontend/backend integration while preserving the existing minimalist builder and landing UX.

## Evidence
- Product postcheck: `src/app/api/engine/chats/[chatId]/product-postcheck/route.ts`, `src/lib/gen/verify/product-postcheck.ts`, and strict schema under `docs/schemas/strict/`.
- Client postchecks: `src/lib/hooks/chat/post-checks.ts`.

## Findings
- Product postcheck adds a server-side quality gate that the chat UI must consume without duplicate error truth.
- Preview prewarm improves perceived latency but must remain opportunistic and non-blocking.
- SEO deploy opt-in must preserve explicit `null` versus `undefined` semantics.
- Version diagnostics must stay connected to version history without redesigning the UI.
- Import validator and strict schemas are backend/pipeline correctness features that reduce generated-site failures.
- Landing scaffold variants must flow through orchestration without changing the wizard’s visual model.

## Recommended Actions
- Keep master API contracts, schemas, rate limits, and validation behavior.
- Preserve local dialog placement, compact copy, tokens, and current builder chrome.
- Document any new product-postcheck or SEO preference fields in the implementation brief.
