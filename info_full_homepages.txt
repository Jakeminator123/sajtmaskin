Vercel Template Directory and MCP Server Ecosystem
Översikt över templates‑katalogen på Vercel

Vercel driver en katalog med templates (mallar) där utvecklare kan kickstarta projekt genom att klona, anpassa och sedan deploya dem på plattformen. Katalogen är tillgänglig via länkar av formen https://vercel.com/templates/<slug> och erbjuder filtrering genom flera filtergrupper. Varje grupp exponeras som en klickbar meny i sidans vänsterspalt och leder till en specifik undersida. De viktigaste grupperna är Use Case, Framework, CSS, Database, CMS, Authentication och Experimentation. När man klickar på ett filter visas en lista med underkategorier; varje underkategori har en egen slug (t.ex. ai, starter, ecommerce).

I sidans body visas mallarna som kort: varje kort innehåller ett <h3>‑element för mallens namn samt en kort beskrivning i ett div‑element med klassen text-label-14. Själva kortet är en länk (<a>), och URL:en pekar mot mallens detaljsida under https://vercel.com/templates/<slug>/<template‑slug>. På detaljsidan finns knappar för Deploy (skapa ett projekt direkt på Vercel) och View Demo (länkar ofta till en live‑preview), samt en länk till källkodens GitHub‑repo.

Navigering och åtkomst till källkoden

När man klickar på ett mallkort öppnas en detaljsida. Där finns:

GitHub‑repo – en länk som går direkt till källkodens GitHub‑repository. Källkoden är oftast öppen och kan klonas eller forkas för anpassning.

För att hitta en malls repo manuellt gör du följande:

Gå till mallen via https://vercel.com/templates/<slug>/<template‑slug>.

Leta efter en länk eller knapp med texten GitHub Repo eller en ikon. Denna länk leder direkt till GitHub‐reposet.

Om sidan inte har en sådan länk kan du ibland scrolla till avsnittet Getting Started där GitHub‑URL:en står.

Vårt Python‑skript (vercel_template_cli.py) kan automatisera detta genom att hämta varje mapps HTML och leta efter den första länken som innehåller github.com/<owner>/<repo>.

Att använda mallarna i egna projekt

Typiskt workflow när du vill använda en Vercel‑mall:

Utforska katalogen och välj en mall som matchar ditt use case.

Klicka på Deploy för att skapa ett nytt projekt på Vercel. Du blir omdirigerad till GitHub där du kan skapa en fork eller importera mallen till ett eget repo.

Klona projektet till din lokala maskin: git clone <repo‑url>.

Installera beroenden med npm install eller pnpm install.

Konfigurera miljövariabler enligt dokumentationen (ex. API‑nycklar eller databas‑URL:er). På mallens sida finns ofta en lista över nödvändiga variabler och hur de ska sättas.

Kör lokalt med npm run dev för att utveckla och anpassa projektet.

Deploya ändringarna. Med Vercel kan du koppla GitHub‑repo till ditt Vercel‑konto så att varje push triggar en deployment.

Vercels officiella MCP‑server och adapter

Vercel har byggt en egen Model Context Protocol (MCP)‑server. MCP är ett öppet protokoll som standardiserar hur AI‑modeller anropar externa verktyg och data; det gör att agentbaserade system kan samtala med t.ex. databaser, CMS eller tredje part via ett gemensamt gränssnitt. Den officiella MCP‑servern finns på https://mcp.vercel.com och ger AI‑verktyg (som Claude, ChatGPT, Codex CLI m.fl.) säkra OAuth‑skyddade rättigheter att nå dina Vercel‑projekt. För att ansluta laddar man ner MCP‑servern i klienten via:

npx add-mcp https://mcp.vercel.com

När du startar klienten (t.ex. ChatGPT‑plugin eller Claude) ombeds du logga in på Vercel och ge åtkomst. Därefter kan AI‑modellen köra verktyg som listor deployments, skapa projekt etc..

Vercel har även publicerat ett MCP‑adapterpaket (tidigare @vercel/mcp-adapter, nu mcp-handler) som gör det enkelt att exponera Node.js‑ eller Next.js‑funktioner som MCP‑verktyg. Adaptern stödjer både Server‑Sent Events (SSE) och Streamable HTTP – den senare introducerades i MCP‑specifikationen i mars 2025 för att ersätta SSE och minska behovet av långlivade anslutningar. Ett enkelt exempel (ett tärningskast‑verktyg) visas nedan:

import { createMcpHandler } from '@vercel/mcp-adapter';
import { z } from 'zod';

export const handler = createMcpHandler(server => {
  server.tool(
    'roll_dice',
    'Rolls an N‑sided die',
    { sides: z.number().int().min(2) },
    async ({ sides }) => {
      const value = 1 + Math.floor(Math.random() * sides);
      return { content: [{ type: 'text', text: `You rolled a ${value}!` }] };
    },
  );
});

export { handler as GET, handler as POST, handler as DELETE };

När man deployar en sådan server på Vercel kan kunder ansluta med Streamable HTTP eller SSE. Artikeln MCP server support on Vercel beskriver hur paketet publicerades i maj 2025 och att det ersätts av mcp-handler. Den nya transporten gör att CPU‑användningen kan halveras när man migrerar från SSE. För äldre klienter kan man använda paketet mcp-remote för att proxya Streamable HTTP över stdio.

Exempel: Next.js MCP‑server med Descope

Vercels katalog innehåller en färdig mall för att köra en MCP‑server integrerad med Descope (en autentiseringsleverantör). Mallen heter MCP with Next.js and Descope och visar hur man:

använder mcp-handler för att definiera verktyg;

använder Descope Node SDK för att validera sessions‑tokens och tillföra auth‑context;

exponerar ett enkelt echo‑verktyg som returnerar text;

sätter upp miljövariabler och kör projektet lokalt via npm run dev;

kräver en giltig bearer‑token för alla MCP‑endpoints, vilket visas i kodsnutten där man hämtar och validerar token och returnerar användar‑scopes.

Detta exempel visar hur man kopplar autentisering och MCP för att bygga säkra verktygsservrar.

Övriga MCP‑servrar och paket

Det finns flera tredjepartsservrar och paket att ladda ner, som levererar färdiga verktyg eller erbjuder mer komplett admin‑funktionalitet:

vercel-mcp-server (Robinson AI Systems) – ett omfattande MCP‑serverpaket publicerat på npm. Paketet innehåller över 50 verktyg för projekt‑ och teamhantering, deployment‑styrning, domän/DNS‑administration, hantering av miljövariabler, edge‑konfiguration och webhookar. Det installeras via npm install @robinson_ai_systems/vercel-mcp och kräver en Vercel access‑token som sätts i .env.

vercel-mcp-server (GitHub) – ett open‑source repo som ger full administrativ kontroll över Vercel via MCP. README visar hur man konfigurerar en Vercel access token och kör servern lokalt eller deployar på Vercel. Verktygsuppsättningen innefattar team‑management, projekt och deployment‑administration samt miljövariabler och roll‑baserat access‑kontroll.

LobeHub AI SDK MCP server – en MCP‑server som exponerar Vercel AI SDK:s kärnfunktioner via MCP och integrerar Figma‑verktyg. Den använder en ToolManager för att organisera verktyg och levereras med en Dockerfile och smithery.yaml för att deploya på Vercel med Smithery.

WorkOS MCP Example – ett exempelrepo som visar hur man bygger en autentiserad MCP‑server med WorkOS AuthKit. (GitHub‑länk, men dokumentation är begränsad.)

Sammanfattning

Vercels templates‑katalog är uppbyggd kring klickbara filtermenyer. Varje filter leder till en undersida där mallar presenteras i kortform. Mallarnas detaljsidor innehåller en länk till GitHub‑reposet och instruktioner för att komma igång. Vercel tillhandahåller även en officiell MCP‑server och adapter för att koppla AI‑modeller till externa verktyg, samt ett ekosystem av öppna MCP‑servrar som kan deployas på plattformen. Genom att kombinera mallar från katalogen och MCP‑servrarna kan man snabbt bygga och distribuera moderna applikationer med AI‑funktioner, autentisering, datalagring och experimentering.
