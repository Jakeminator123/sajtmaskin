## Master Pipeline Changes
- `deriveBuildSpec` controls change scope, preview policy, quality target, verification policy, context policy, style pack, token budgets, references, capabilities, and route realization.
- `buildOrchestrationContract` defines `requiredRoutePaths` and `generationToValidate`.
- Route realization supports `primary-full-with-shells`, full routes, companion auth routes, and deferred extra routes.
- Capability detection feeds heavy flags and verification/context policy.
- F3 remains explicit through lifecycle/override.
- Website-to-app promotion is guarded on follow-up.

- F3 only by explicit integrations lifecycle or override.
- P26 intent-promotion guard for persisted non-app scaffolds.
- Required-route subset semantics for shell mode.
- Capability flags stay synced with heavy capability keys.
