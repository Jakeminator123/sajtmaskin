Länkens åtkomst
Chattlänken (t.ex. v0.app/chat/nano-banana-pro-playground-…) verkar vara en delad (“unlisted” eller “public”) chatt vilket innebär att alla med länken kan se den utan inloggning. Enligt v0:s dokumentation kan en “Unlisted” chatt nås av vem som helst med länken, och “Public” är helt öppen[1]. Om chatten däremot ställts in som privat krävs inloggning och inbjudan för att få åtkomst.
API-interaktion och kontonivåer
Ja – chattkonversationen kan hanteras via v0 Platform API. Man kan till exempel använda v0.chats.sendMessage för att skicka nya meddelanden och v0.chats.getById för att hämta historiken. Detta kräver bara ett v0-konto med giltig API-nyckel; både gratis- och betalda planer har API-åtkomst (gratisplanen har dock låg användningsgräns, t.ex. 7 meddelanden per dag[2]). Team-/Enterprise-planer krävs endast om ni vill dela chatt och samarbeta inom ett arbetslag – Team-planen ger möjligheten att ”share chats and collaborate”[3].
Inbäddning (iframe)
Det går att bädda in v0-appens resultat-URL i en <iframe>. Exempelvis returnerar ett anrop till v0.chats.create() ett fält chat.demo som är en URL till det genererade gränssnittet, vilken kan användas som src i en iframe[4]. Det är dock främst själva applikationens förhandsgranskning som bäddas in på det viset. Att direkt bädda in själva chattgränssnittet via iframe är inte dokumenterat och kräver sannolikt inloggning (eller är helt enkelt inte avsett för inbäddning).
Använt API bakom kulisserna
V0:s chattgränssnitt drivs av v0 Platform API, inte endast det fristående Model-API:t. Model API ger direkt åtkomst till AI-modeller för kodgenerering, medan Platform API ger komplett utvecklingsinfrastruktur – inklusive chatt-hantering, filoperationer och exekvering – bakom v0.dev’s gränssnitt[5][6]. Med andra ord används Platform API för samtalshantering (skapa chatt, skicka och hämta meddelanden) och generera/applikationshantera koden i bakgrunden.
Återskapa från egen webbplats
För att återskapa detta i egen app använder man v0 Platform API. Till exempel kan man anropa v0.chats.create({ message: 'Din startprompt…' }) för att skapa en ny chatt[7]. Därefter kan man fortsätta konversationen med upprepade v0.chats.sendMessage({ chatId: chat.id, message: '…' }). Minimikravet är ett v0-konto – även gratisplanen kan användas för att prova (med de begränsningar som anges[2]). För större volymer eller team-samarbete rekommenderas Premium- eller Team-planer (Premium ger högre resurser och fler meddelanden, Team-planen möjliggör att dela chatten inom gruppen[2][3]).
Källor: v0:s officiella dokumentation och resurser för delning, API och prissättning[1][4][5][6][7][2][3].
