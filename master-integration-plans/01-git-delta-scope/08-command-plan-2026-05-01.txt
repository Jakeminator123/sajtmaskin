## Findings
- A dirty tree makes `git merge origin/master` unsafe and hard to review.
- Planning/review artifacts and generated `data/runs/**` should not be mixed with product merge commits.
- Full verification should include typecheck, lint, and tests after any merge or conflict resolution.
- Push is optional and must be non-force unless explicitly approved.

## Recommended Actions
- Stop after read-only mapping until user approves integration.
- Choose either scoped commits or `git stash push -u` before merge.
- Prefer scoped commits over a single opaque WIP commit when time allows.
- Never run browser verification against the same live builder chat the user is using.
