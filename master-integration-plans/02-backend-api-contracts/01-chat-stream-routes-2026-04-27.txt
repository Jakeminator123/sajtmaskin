## Contract Risks
- Ordning: `meta.versionId` vs `preview-ready` — fel binding till heartbeat om ordning ändras.
- `done` efter `error` utan recovery — tom done bryter felväg.
- Builder coexistence — två streams mot samma `chatId`.
- Init vs följd: content-type måste vara `text/event-stream` när klienten förväntar SSE.
- Nya event från master — antingen nya `case` i `stream-handlers` eller tysta via kontrakt.

## Required Adaptations
- Nya event: `builder-stream-contract.ts` + `stream-handlers` om UI reagerar.
- Ändrad `done`/`error` HTTP: synka `stream-handlers`, `post-checks`, `helpers`.
- Tunn `route.ts` ska fortsatt delegera till lib så test-URL:er hålls.
- `engineBaseVersionIdOverride` / autofix: behåll genom `useSendMessage`.
