# User-site preview: degraded / placeholder environment variables — policy
#
# Purpose
# -------
# Explain why and how to inject *non-secret placeholders* into **generated user
# projects** (not the Sajtmaskin app) so preview / sandbox can start without
# real integrations.
#
# Canonical KEY=value list (dotenv-style)
# ---------------------------------------
# Edit and copy from:
#   **config/ai_models/40-generated-site-integration-placeholders.env.txt**
#
# That file is the same idea as `config/prompt-static/*.md` fragments: plain
# text under `config/`, indexed from **config/ai_models/manifest.json** under
# `generatedSiteIntegrationPlaceholders` so tooling and agents have one place to look.
#
# Node helper (read file from disk — server/scripts only, not client bundles):
#   **src/lib/ai-models/load-generated-site-placeholders.ts**
#
# Alignment with codegen (system prompt)
# ----------------------------------------
# - config/prompt-static/11-behavioral-rules.md: mock data by default; real
#   backends only when the prompt clearly requires them; document required env
#   vars (e.g. in a comment at the top of the file).
# - Generated apps should treat missing or placeholder secrets as *degraded mode*:
#   UI and static routes render; side effects no-op or show an in-UI message —
#   never crash the dev server on import.
#
# Section B/C notes (Redis, SQL, Auth0)
# -------------------------------------
# If generated code creates Redis/DB clients at module scope and connects
# immediately, placeholders may still cause timeouts. Prefer lazy clients or
# skip init when URL is missing or equals a known preview sentinel. For
# Postgres-shaped URLs, empty `DATABASE_URL` often breaks migrate/build; the
# fragment file supplies local-shaped URLs for preview only.
#
# This policy file is *not* loaded automatically by the Sajtmaskin app.
# Auto-injection of placeholders into preview pipelines is optional product
# work; the fragment + manifest are the repository contract.
#
