Rekommendationer för enkla API:er och resurser till React-sidor
====================================================

Syfte
-----
Det här dokumentet samlar enkla, praktiska API:er och resurser du kan använda direkt från en React-sida (eller via en enkel backend) för att snabbt få avatarer, loggor, bilder, diagram, QR-koder m.m.

1. Avatarer & enkla ”loggor”
----------------------------
**DiceBear**

- Typ: HTTP-API (ingen API-nyckel krävs).
- Vad: Genererar avatarer och enkla ”loggor” som SVG/PNG baserat på en seed-text.
- Bra till:
  - Profilbilder (t.ex. seed = användarnamn, e-post).
  - Små logoikoner med initialer.
- Exempel (avatar):
  - Bas-URL: https://api.dicebear.com/9.x/avataaars/svg?seed=DIN_SEED
  - Seed kan vara t.ex. ”jakob”, ”user@example.com” osv.
- Exempel (initialer/logo):
  - Bas-URL: https://api.dicebear.com/9.x/initials/svg?seed=JP

För React kan du helt enkelt använda URL:en i ett `<img>`-element.

2. Placeholder- & bakgrundsbilder
---------------------------------
**Lorem Picsum**

- Typ: HTTP-API (ingen nyckel).
- Vad: Returnerar slumpmässiga foton i valfri storlek.
- Bra till:
  - Test-kort / cards.
  - Bakgrundsbilder medan du skissar UI.
- Exempel:
  - Slumpad bild: https://picsum.photos/400/250
  - Bestämd bild med seed: https://picsum.photos/seed/poker/400/250

**Unsplash API**

- Typ: Kräver gratis konto + API-nyckel.
- Vad: Sök efter riktiga foton (”poker”, ”city night”, ”space” osv.).
- Rekommenderat arbetssätt:
  - Anropa Unsplash från backend (Node/Python).
  - Frontend pratar med din backend, inte direkt med Unsplash.
- Bra till:
  - Snygga hero-bilder.
  - Tematiska bakgrunder (t.ex. spel, stad, natt, teknologi).

3. Diagram som bilder
---------------------
**QuickChart**

- Typ: HTTP-API som genererar diagram som bild (PNG).
- Vad: Du skickar ett Chart.js-liknande config-objekt som query-param.
- Fördel:
  - Ingen tung chart-lib i frontenden, du får bara en färdig bild-URL.
- Bra till:
  - Snabb visualisering av statistik, t.ex. EV, vinstkurvor, händer per dag osv.

Exempelidé (bar chart-konfiguration i pseudo):

- Skicka JSON:
  - type: "bar"
  - data.labels: ["Mån", "Tis", "Ons", ...]
  - data.datasets[0].label: "EV (bb/100)"
  - data.datasets[0].data: [2, -3, 5, 1, 4]

- URL-struktur:
  - https://quickchart.io/chart?c=<URL-encodat JSON>

I React använder du bara URL:en som `src` i ett `<img>`.

4. QR-koder
-----------
**QuickChart QR**

- Typ: HTTP-API för QR-koder.
- Vad: Genererar en QR-bild från text/URL du skickar in.
- Exempel:
  - Bas-URL: https://quickchart.io/qr?text=<URL-encodad text>
  - `text` kan vara vilken sträng eller URL som helst.

Bra användningar:
- QR-kod till en profilsida.
- QR-kod för att dela en specifik route i din React-app.
- Visas enkelt i ett `<img>`-element.

5. AI-genererade loggor & illustrationer
----------------------------------------
**Bild-API:er (t.ex. OpenAI / DALL·E-liknande)**

- Tanke: Använd ett bild-API via backend för att ta fram:
  - Minimalistiska loggor.
  - Små illustrationer/ikoner.
  - Bakgrundsbilder som matchar ditt tema (poker, tech, futurism, etc.).
- Rekommenderat arbetssätt:
  1. Skriv ett litet script (Python/Node) som anropar bild-API med en tydlig prompt, t.ex.:
     - "Minimalistisk logo för en poker-tracker, neonblå, mörk bakgrund, PNG med transparent bakgrund"
  2. Spara bilden lokalt eller i ett CDN.
  3. Använd bilden i din React-app som en vanlig `.png`/`.webp`.

Detta är mer ”build step” än live-API i frontenden, men extremt praktiskt för att snabbt få fram varianter på loggor.

6. Övriga användbara UI-resurser (ej rena HTTP-API:er)
------------------------------------------------------
**Ikonbibliotek**

- `react-icons`: Massor av ikonpaket (Font Awesome m.m.) som React-komponenter.
- `lucide-react`: Tunna, snygga linjeikoner.

Varför:
- Snabbt att få snygga ikoner till knappar, menyer osv.
- Bra för att visualisera olika delar av appen (pokerbord, stats, historik, inställningar).

**Lottie-animationer**

- Idé: Små vektoranimationer (JSON-filer) som spelas upp i React via t.ex. `lottie-react`.
- Bra till:
  - Loading-indikatorer.
  - “Tomma” states (t.ex. ”Inga händer hittades”).
  - Små micro-interactions när något lyckas/misslyckas.

7. Praktiska tips när du använder dessa i React
-----------------------------------------------
1. Håll API-nycklar borta från frontenden
   - Använd backend (Node/Python) eller serverless functions för:
     - Unsplash
     - AI-bild-API:er
   - Lägg nycklar i `.env` och exponera aldrig direkt i klienten.

2. Cacha URL:er
   - För avatarer, diagram m.m. kan du spara den genererade URL:en i t.ex.:
     - Local storage
     - Din databas (kopplad till användaren)
   - Då slipper du att samma sak genereras helt i onödan varje gång.

3. Använd alltid `alt`-texter
   - För bilder och diagram:
     - Beskriv kort vad användaren ser (”Avatar för Jakob”, ”EV per dag”, ”QR-kod till min profil”).
   - Bra för tillgänglighet och SEO.

4. Håll komponenterna små & återanvändbara
   - Skapa egna wrapper-komponenter i React, t.ex.:
     - `<Avatar seed="..." />`
     - `<RandomImage topic="poker" />`
     - `<StatsChart data={...} />`
     - `<QrCode value="..." />`
   - Då kan du byta implementation (t.ex. byta API) utan att uppdatera hela appen.

5. Tänk på laddningstid
   - Bilder kan bli stora (särskilt Unsplash/AI).
   - Använd:
     - Rimliga dimensioner (t.ex. 400x250 istället för 4000x2500).
     - Eventuellt `loading="lazy"` på `<img>` i React.

Sammanfattning
--------------
Med bara ett par enkla URL-baserade API:er kan du snabbt få:

- Avatarer och enkla loggor (DiceBear).
- Placeholder- och bakgrundsbilder (Lorem Picsum, Unsplash).
- Diagram som bilder (QuickChart).
- QR-koder (QuickChart QR).
- AI-genererade loggor/illustrationer (via bild-API i backend).
- Plus ikoner och animationer (react-icons, lucide-react, Lottie).

Bygg små, tydliga React-komponenter runt dessa så får du ett flexibelt UI-kit som du kan återanvända i massor av projekt.
