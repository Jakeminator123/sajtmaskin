# User-site preview: degraded / placeholder environment variables
#
# Purpose
# -------
# Reference list for injecting *non-secret placeholders* into generated user
# projects so a full Next.js preview (iframe, sandbox, or similar) can start
# without real integrations. Values are intentionally fake; they must not
# grant access to any service.
#
# Alignment with codegen (system prompt)
# ----------------------------------------
# - config/prompt-static/11-behavioral-rules.md §7–8: mock data by default;
#   real backends only when the prompt clearly requires them; document required
#   env vars (e.g. in a comment at the top of the file).
# - Generated apps should treat missing or placeholder secrets as *degraded
#   mode*: UI and static routes render; side effects (email, DB writes, payments)
#   no-op or show a clear in-UI message — never crash the dev server on import.
#
# This file is *not* loaded automatically by the app. It is documentation +
# a copy-paste / merge source for preview runners and future tooling.
#
# -----------------------------------------------------------------------------
