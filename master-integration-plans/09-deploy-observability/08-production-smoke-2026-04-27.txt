## Production Changes In Master
- **E2E deploy (Playwright):** `npm run test:deploy-smoke:e2e` — `e2e/deploy/*.smoke.spec.ts`, `SAJTMASKIN_E2E_BASE_URL` + `SESSION_COOKIE` + `..._DEPLOY_CHAT_ID` + `..._DEPLOY_VERSION_ID` (alla fyra = annars **skip**).

- **Handoff preview → prod:** se `docs/architecture/fas3-preview-and-deploy.md` — samma `versionId`/filer, `precheck`, `DEPLOY_MISSING_ENV`, `repair_available` blockering, separata verify/preview-lanes.

## Local Behavior To Preserve
- `useBuilderDeployActions` kan pinga `/api/health` före deploy-UX — bryt inte 200/JSON form.
- Precheck utan kreditförbruk; full deploy endast när förväntat.
- Opt-in e2e: CI kan gå “skip” medvetet — ersätt manuell staging-checklista.

## Security / Secret Risks
- E2E env-variabler (session cookie, chatId) — **dokumentera** som secrets i CI, aldrig commit.
- Publika health kan läcka feature-flags; accepterat; överdrivna detaljer i error body = håll kort.
