Task:
Build a read-only Prompt Routing Inspector in backoffice.

Goal:
Given a prompt, show why the builder would choose scaffold, variant, route-plan, capabilities and dossiers.

Must show:
- prompt
- buildIntent used
- scaffold keyword scores
- scaffold embedding candidates if available
- whether embedding override happened
- matched game patterns
- matched veto patterns
- inferred capabilities
- follow-up capabilities if relevant
- selected dossiers
- route plan
- explicit page-count detection
- selected scaffold variant
- warnings from lifecycle/compatibility checks if available

Constraints:
- Read-only only.
- Do not change routing behavior.
- Reuse existing runtime functions where possible.
- If Python backoffice cannot import TS directly, create a small JSON/debug endpoint or script rather than duplicating logic.
- No new dependencies unless absolutely needed.

Useful prompts to test manually:
- "Bygg ett Pac-Man spel"
- "Gaming news blog"
- "Sajt för en tv-spel butik"
- "Landing page med 3 sidor: hem, om oss, kontakt"
- "Lägg till en ai-chatt på en ny sida"
- "Flytta pricing section ovanför FAQ"

Deliverable:
- Backoffice page or panel
- JSON export of routing trace
- At least one test for trace serializer if practical
