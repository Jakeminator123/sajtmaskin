SYFTE
-----
Den här filen samlar beslut, risker, pipeline och exakta steg så att ingen
råkar blanda ihop:
  - v0-mallar (produkt / Mall-fliken)
  - runtime scaffolds (src/lib/gen/scaffolds, scaffold-embeddings.json)
  - template-library (Vercel-katalog → kurering → template-library.generated.json
    + template-library-embeddings.json)
  - Playwright-specen som skrapar vercel.com/templates

Trötthet och förkortningar ursäktas inte i koden; här skriver vi ut allt
pedantiskt.

2. VARFÖR "SPOEKMAPPEN" VAR PROBLEMATISK
----------------------------------------
Bra med gitignore på en rotmapp:
  - Undviker att committa skrap, experiment, personliga anteckningar.

Dåligt när package.json pekar BARA dit:
  - Ny git clone har ingen mapp → npm run references:discover* faller.
  - Två kopior av samma spec (e2e + vercel_templates_levels) → drift om en
    uppdateras.

Lösning vi genomfört:
  - Kanonisk spec under e2e/ (spårad).
  - package.json pekar på e2e-sökvägen.
  - vercel_templates_levels/ förblir ignorerad som valfri spillra; kan raderas
    lokalt.


3. SCAFFOLDS — VAD FÅR INTE GÅ SÖNDER
-------------------------------------
Playwright-specen ändrar INTE:
  - scaffold-embeddings.json (genereras av scripts/generate-scaffold-embeddings.ts
    från befintliga manifest).
  - Mall-flödet templates:* (v0).

Den matar template-library-kedjan via raw-discovery:
  - Efter scrape: kör befintliga npm-steg för import / hydrate / build /
    template-library:embeddings enligt research/external-templates/README.md.
  - Först då uppdateras committade JSON som searchTemplateLibrary läser.

Promote-to-scaffold och liknande skript kopplar template-library entries till
scaffolds — det är ett MEDVETET steg, inte automatiskt vid references:discover.

Om du bara kör Playwright utan build/embeddings får du NY rådata på disk men
GAMMAL runtime i src/lib/gen/template-library/*.json tills du kör pipelinen.


4. PIPELINE (ORDNING)
---------------------
A) references:discover*  → raw-discovery/current/*
B) template-library:import (eller motsvarande för er dataset-label)
C) template-library:hydrate-cache (vid behov)
D) template-library:build
E) template-library:embeddings
F) (Valfritt) kurering, promote-to-scaffold, generate-scaffold-embeddings — när
   ni medvetet vill uppdatera scaffolds.

Verifiering: se template-library-external-integration.test.ts och
scaffold-embeddings-parity.test.ts för vad som förväntas committat.

6. BRA / DÅLIGT — SAMMANFATTNING
--------------------------------
Bra:
  - Clone + npm install → discover-scripts hittar filen.
  - Tydlig separation: template-library vs scaffold-embeddings dokumenterad.
  - Ignorerad legacy-mapp stör inte git om den finns kvar lokalt.

Dåligt / kvar att bevaka:
  - DOM på vercel.com kan bryta specen.
  - Fler entries i katalogen utan bra kurering kan öka brus i embedding-sök.
  - Om någon återintroducerar package.json-pekning till enbart ignorerad mapp
    återkommer spökproblemet.


7. VERIFIERING (LOKALT)
-----------------------
  npx playwright test e2e/vercel-templates/scrape-catalog.spec.ts --list
  npm run typecheck

references:discover kräver nät och Chromium; kör inte i CI som standard om ni
inte sätter upp det medvetet.
